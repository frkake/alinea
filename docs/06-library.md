# 06. ライブラリと進捗管理

> 進捗管理は「積読の罪悪感を生む機能」になりやすい。管理のための管理を要求せず、読む行動から自然に状態が育つ設計にする(P6)。ライブラリ(1e/4a)・ダッシュボード(1d)・コレクション(4b/4c)・通知(4a)・横断検索(1e/4e)・読了フロー(1g)は確定デザインを正とする。

## 1. ステータス体系

読書ステータスは 6 値。各ステータスは固有のドット色を持ち、ピル(ドット+ラベル)で常に同じ色表現を使う(1e/4a)。

| ステータス | 色 | 意味 | 入り方 |
|---|---|---|---|
| 読む予定 | `#9AA0A6`(グレー) | いつか読む。既定の着地点 | 取り込み時の既定値 |
| すぐ読む | `#C49432`(アンバー) | 短期キュー。「次に手に取る数本」。ダッシュボードの「すぐ読むキュー」に載る | 手動。キューが 6 本になると控えめに警告(§6.2。ブロックはしない) |
| 読んでいる | `var(--pr-a)`(アクセント) | 着手済み。進捗バー・前回位置を表示 | 手動+自動遷移提案(§2) |
| 読んだ | `#659471`(グリーン) | 読了。読了フロー(§3)が走る | 手動+提案 |
| あとで再読 | `#8E7AA6`(パープル) | 読了したが再読が必要 / 途中だが今は寝かせる | 手動 |
| 保留 | `#B0ACA2`(淡グレー) | 読むのをやめた・優先度が消えた | 手動 |

- 遷移は制約しない(どの状態からどの状態へも変更可)。
- 補助属性: 優先度(**高/中/低**。「高」は警告色 `#A05A42` で強調表示)、締切(任意。警告色チップ「締切 7/16」)、理解度(1–5)、重要度(**低/中/高**。§3)。
- **クイックフィルタ 5 種**(1e/4a): 「すべて / 未読 / 途中 / 読了 / 要再確認」を件数付きピルで表示する。ステータスからの合成定義:
  - 未読 = 読む予定 + すぐ読む
  - 途中 = 読んでいる + 保留
  - 読了 = 読んだ
  - 要再確認 = あとで再読
  - デザインの件数例「すべて 41 / 未読 12 / 途中 4 / 読了 23 / 要再確認 2」(12+4+23+2=41)とこの合成は整合する。

## 2. 自動遷移の方針: 「変更せず、提案する」

ステータスの自動変更は既定ではしない(勝手に変わる管理は信頼されない。P6)。代わりに提案する:

- 「読む予定/すぐ読む」の論文を **3 分以上** アクティブに読んだら、通知(§7)で「✦ ○○ を 3 分以上読んでいます。**「読んでいる」にしますか?**」を 1 回だけ出す。「変更する / そのまま」の 2 択+補足「ステータスは勝手に変わりません — 提案のみ(設定で変更可)」(4a 逐語)。
- 最終セクション付近まで到達したら「『読んだ』にしますか?」を同様に提案。
- **設定は 3 段階**(設定画面 4f「読書の計測と提案」): **自動適用 / 提案する(既定)/ 提案しない**。

## 3. 読了フロー(1g)

ステータスを「読んだ」に変更すると、中央モーダルの軽量ダイアログ(幅 460px、背景スクリム)を出す。**全項目スキップ可・入力を強制しない**。

- ヘッダ: ✓「「読んだ」にしました」+「読了日 2026-07-06 · 累計読書時間 3時間12分(自動記録)」— **読了日と累計読書時間は自動記録の表示のみ**で編集 UI はない。
- **理解度**: 1〜5 の 5 段階ドット選択。選択値にラベルを併記(表示形式「n/5 — ラベル」。デザイン逐語は「4/5 — だいたい追えた」)。
  - 決定: デザインに描かれているラベルは 4 のみのため、残りを暫定定義する(1=「ほぼ分からなかった」/ 2=「ところどころ分かった」/ 3=「半分くらい追えた」/ 4=「だいたい追えた」/ 5=「完全に理解した」)。理由: UI 上全値にラベル表示が必要なため。文言は実装時に調整可。
- **重要度**: 「低 / 中 / 高」の 3 値セグメント(旧仕様の 1–3 数値表現を廃止し、デザインのラベル表現に統一)。
- **ひとことメモ**: 単一の自由記述。ラベル補足「— 何に使えるか」を入力ガイドとして表示。
- 導線カード 2 枚(チャット→メモ昇格・読み返しへの導線):
  - 「**✦ 要約をメモに保存**」— 説明「チャットの「詳細要約」を根拠つきで保存」([05-chat.md](05-chat.md))
  - 「**記事モードで読み返す →**」— 説明「メモとチャットから読み物を自動構成」([07-figures-and-articles.md](07-figures-and-articles.md))
- フッタ: 「**すべてスキップ**」(左・テキストリンク)/「保存」(右・プライマリ)。「×」でも閉じられる。

## 4. タグとコレクション

役割を分ける:

- **タグ**: 横断的なラベル(手法名・テーマなど任意軸)。フラット・入力補完あり。arXiv カテゴリ等はシステムが承認式で提案する(「提案: cs.CV +」チップ。[02-ingest.md](02-ingest.md))。
- **コレクション**: 手動キュレーションの **順序付き** リスト。プロジェクト・輪読会・講義用。1 論文は複数コレクションに入れられ、**並び順は共有ページにも反映される**(4b フッタ逐語)。

### 4.1 コレクション詳細画面(4b)

| 機能 | 仕様(デザイン準拠) |
|---|---|
| 順序 | 「発表順 — ドラッグで並べ替え」。⋮⋮ ハンドル+順序番号バッジ。「+ 論文を追加」で追加 |
| 締切 | 締切日+残日数バッジ「締切 7/16 — 残り 10 日」。サイドバーのコレクション項目にもミニバッジ(7/16)を併記 |
| 説明文 | 自由記述。「説明を編集」でインライン編集 |
| 進捗集計 | 読了進捗バー(緑 `#659471`)+「3/5 読了」(ステータス「読んだ」の本数/総数) |
| 担当者割当 | 論文ごとに発表担当を割当。自分担当は「担当: 自分」ピル(アクセント地・白字)で強調、他メンバーはサブ行に「担当: 佐藤」等 |
| 発表時間メタ | 論文ごとに「発表 25 分」等の任意メタ。「予備(時間があれば)」のような注記も可 |
| 行の可変表示 | 未着手=「未着手」(警告色)/ 読んでいる=「§2.1 · 42%」ミニ進捗 / 読んだ=「理解 4/5」/ 未評価=「—」 |
| CTA | **未着手かつ自分担当** の行はハイライト(--pr-as 背景)し「読み始める」ボタンを表示 |

- 決定: 担当者はコレクション内のメンバーシップ属性(表示名文字列+「自分」フラグ)であり、グループ/アカウント機能ではない(00-product.md Q4 の暫定判断どおり。デザインに担当者のアカウント連携 UI はない)。

### 4.2 共有リンク管理(4b 右上カード)

- 「閲覧用共有リンク」をコレクション単位で発行できる。URL 形式は `alinea.app/c/{token}`(例: `x8Kf3qPw`)。
- 状態バッジ「発行済み」/「リンクを無効化」で失効。「コピー」で URL コピー、「共有ページを確認 →」でプレビュー。
- 属性表示: 「閲覧のみ · アカウント不要 · noindex」。
- 「**共有ページにメモを含める**」トグル(ON 時は含まれるメモ件数「1 件」を表示)。

## 5. 共有ページ(4c)

コレクション共有リンクの受け手が見る、**v1 唯一の共有面**(記事の公開は v2)。

- **アカウント不要・閲覧専用・noindex**。編集・並べ替え UI は一切出さない。フッタで再掲: 「このページは閲覧専用です · アカウント不要 · 検索エンジンには登録されません(noindex) · メモは共有者が許可したもののみ表示」。
- ヘッダは縮退構成: ロゴ+モードバッジ「共有されたコレクション — 閲覧専用」+誘導文言「自分のライブラリで論文を読むには」+CTA「**Alineaをはじめる**」。
- 表示内容: コレクションのタイトル・説明文・メタ行(「YK さんが共有 · 更新 2026-07-06 · 5 本 ·」+締切バッジ)+番号付き論文カード(順序保持)。
- 各カード: タイトル / 著者・会議年 / 「arXiv ↗」外部リンク / ✦ 付き要約(任意)/ 「共有者のメモ」ボックス(§4.2 のトグルで許可したもののみ)。

## 6. ダッシュボード(ホーム画面 1d)

ログイン直後の画面。デザイナー注記どおり「「次に何を読むか」に 5 秒で答える」。構成は 5 つ:

### 6.1 続きを読む
「読んでいる」の直近 **最大 3 件** をカード表示。サムネイル+タイトル+書誌+読書進捗バー+「前回: §2.1 整流フロー · 昨日」(前回位置セクション+相対時刻)+「**再開 →**」で前回位置からリーダーを再開。

### 6.2 すぐ読むキュー
「すぐ読む」ステータスの論文の手動順序付きキュー。見出しに「6 本 · ドラッグで並べ替え」。

- 各行: ⋮⋮ ドラッグハンドル+順位番号+タイトル+著者・年+優先度バッジ(高=警告色/中・低=グレー)+締切バッジ(あれば)。
- **積みすぎ警告**: キューが 6 本になると黄系バナー「**キューが 6 本になっています — 積みすぎかも?**」+「整理する」アクション+「×」で閉じる(旧仕様の「5 件を超えると促す」をデザインの文言・閾値に置き換え。6 本以上で表示)。

### 6.3 締切が近い
締切は **2 階層** で扱う:

1. **コレクション単位**: 名前+「残り 10 日」バッジ+「コレクション · 5 本中 3 本読了」+緑の読了進捗バー。
2. **論文単位**: タイトル+「担当発表 · 締切 7/16 · 未着手」(未着手は警告色強調)。

### 6.4 最近追加
今週の取り込み(「今週 6 本」)を 2 列カードで表示。取り込みパイプラインの状態がカードに出る([02-ingest.md](02-ingest.md)):

- 処理中: 「✓ 書誌」「✓ アブスト訳・要約」「本文翻訳中 68%」+進捗バー+「§3 まで読めます · 開いたセクションを優先翻訳」+「読み始める」ボタン(**部分読書可**)。
- 完了: 「✓ 翻訳完了 — 読める状態です」。
- **✦ 3行要約**(①②③形式)と確定タグ+**提案タグチップ**(「提案: cs.CV +」「solver +」— クリックで承認)。
- ソースバッジ: 「A」(arXiv 系・アクセント淡色)/「B」+「PDF 取り込み」(グレー)。

### 6.5 統計(控えめに)
「今週」カードのみ: 読了本数(「3 本 読了」)・読書時間(「4.2 時間」)・直近 12 週の読書時間棒グラフ(今週のみアクセント色)・「詳細 →」。連続記録(streak)などのゲーミフィケーションはしない(P6)。

## 7. 通知センター(4a)

ヘッダの通知アイコン(グリフ「◷」)+ポップオーバー。未読があるとアイコン右上に **未読ドット `#C49432`**(6px)を表示する。

- ポップオーバーヘッダ: 「通知」+「**すべて既読にする**」。
- 通知項目は未読(淡背景+アクセントドット)/既読(白背景・ドットなし)の 2 状態。タイムスタンプは相対表示(「昨日 19:40」「今日 8:00」)。
- **通知は 3 種類**:

| 種類 | 本文例(逐語) | アクション |
|---|---|---|
| 翻訳完了 | 「**翻訳が完了しました** — Stochastic Interpolants: …」 | 「読み始める →」で該当論文を開く([03-translation.md](03-translation.md)) |
| ステータス変更提案 | 「✦ InstaFlow を 3 分以上読んでいます。**「読んでいる」にしますか?**」 | 「変更する / そのまま」ボタン。§2 の提案設定に従う |
| 締切リマインド | 「輪読会 2026-07 の締切まで **10 日** — 未着手 1 本」 | 「コレクションを開く →」 |

- AI 起点の提案通知には ✦ マークを付ける(P1)。

## 8. ライブラリ画面(1e テーブル / 4a カード)

### 8.1 共通シェル
- トップバー: ロゴ+グローバル検索バー(プレースホルダ「ライブラリ全体を検索 — 本文・訳文・メモ・チャット」、ショートカット ⌘K)+通知+アバター。
- サイドバー: ホーム / ライブラリ / **語彙帳**([11-vocabulary.md](11-vocabulary.md)。件数バッジ付き)/ コレクション(締切ミニバッジ+件数。「+ 新規コレクション」)/ 保存フィルタ / フッタ「設定 · エクスポート」。
- 見出し行: タイトル+総数(「41 本」)+ビュー切替セグメント「**カード / テーブル**」+「並び: 更新日 ▾」+(テーブル時)「**この条件を保存**」ボタン。

### 8.2 カードビュー(4a)
サムネイル(論文の図)+ソースバッジ+タイトル(2 行クランプ)+著者・会議年+✦ AI 要約(2 行クランプ)+フッタ(ステータスピル/タグ/締切チップ/右端メタ)。

- 「読んでいる」カードのみ読書進捗バーを表示。読了カードは「理解 4/5 · 5.1h」を表示。その他は累計時間または「優先: 高/中」。

### 8.3 テーブルビュー(1e)
**列は 10 列固定**(旧仕様の「列カスタマイズ」は廃止 — デザインに列編集 UI がなく、固定列で網羅されているため):

`✓(選択) / 論文(サムネ+タイトル+著者·出典) / ステータス / 品質 / タグ / 優先度 / 締切 / 読書時間 / 理解度 / 追加日`

- 品質バッジ: A(アクセント淡色)/ B(グレー)([02-ingest.md](02-ingest.md) の 2 段階)。
- PDF アップロード由来の論文はタイトル末尾に「**PDF 取り込み**」バッジ+出典「アップロード」で区別する。
- 未設定値は「—」。締切・優先度「高」は警告色 `#A05A42` の太字。
- ソートはツールバー「並び: 更新日 ▾」と列ヘッダクリック(「論文 ↑」昇順インジケータ)の両方から行える。

### 8.4 フィルタと保存フィルタ
- クイックフィルタピル: §1 の 5 種(件数付き)。
- **属性フィルタ 5 種**のドロップダウン: 「ステータス ▾ / タグ ▾ / コレクション ▾ / 品質 ▾ / 年 ▾」。適用中フィルタはアクセント色チップ「タグ: distillation ×」で表示し × で解除(旧仕様にあった venue フィルタはデザインに存在しないため「年」までとする。venue は検索で拾える)。
- **保存フィルタ(スマートリスト)**: 現在のフィルタ+ソート条件を「**この条件を保存**」で名前をつけて保存し、サイドバーの「保存フィルタ」に件数付きで固定(例: 「締切あり 3」「cs.CV の未読 7」「高優先度 4」)。

### 8.5 一括操作
行チェックボックス(+ヘッダ全選択)で複数選択すると、選択行がハイライトされ、画面下部中央に **フローティング一括操作バー**(ダーク地)が出現する:

「**2 件を選択中** | ステータス変更 ▾ / タグ追加 / コレクションへ | 選択解除 ×」

- 決定: 複数選択 UI はテーブルビューにのみ描かれているため、一括操作はテーブルビューの機能とする(カードビューからはテーブルに切り替えて行う)。

## 9. 横断検索(1e ドロップダウン / 4e 全結果画面)

S5(想起)を支える中核機能。対象は **本文(原文・訳文)・メモ・注釈・チャット履歴・記事**+書誌。

### 9.1 検索ドロップダウン(1e)
- ⌘K または検索バークリックで起動。入力中にインクリメンタルにプレビューを表示、esc で閉じる(「esc で閉じる」ヒント表示)。
- ヘッダ: 「「EMA teacher」の結果 12 件」+「本文・訳文・メモ・チャット・記事を横断」。
- 結果プレビューは上位 3 件(種別バッジ+タイトル+位置メタ+検索語ハイライトつきスニペット+遷移リンク)+フッタ「**すべての結果を表示(12 件)→**」で全結果画面(4e)へ。

### 9.2 全結果画面(4e)
専用の全画面レイアウト(トップバー+左ファセット+結果リスト)。

- ファセット 2 軸(件数付き): 「ヒット源」= すべて / 本文(原文・訳文)/ メモ・注釈 / チャット履歴 / 記事、「論文で絞る」= 論文別。
- 結果サマリ「「EMA teacher」の結果 12 件 · 3 論文」+「並び: 関連度 ▾」。
- 結果は **論文(または記事)単位でグループ化**: グループヘッダにサムネ+タイトル+著者・会議+読書ステータスピル+グループ内件数。
- **ヒット源バッジ 4 色**: 本文(原文/訳文)=アクセント色(`--pr-as` 地 + `--pr-a` 文字)/ チャット=紫 `#6E5A7E` / メモ=緑 `#4C7458` / 記事=グレー `#777B81`。検索語は琥珀色 `rgba(196,148,50,0.30)` の mark でハイライト。スニペットのフォントは源に応じて切替(原文=Source Serif 4 / 訳文=Noto Serif JP / その他=UI フォント)。
- **源別の遷移**: 本文→「該当位置へ →」(ビューアの該当アンカー)/ チャット→「スレッドを開く →」/ メモ→「メモを開く →」/ 記事→「記事モードで開く →」。
- **日英クロス検索**: 「日本語クエリは訳文に、英語クエリは原文にヒットします(クロス検索)」(4e 逐語)。同一ブロックの原文ヒットと訳文ヒットは **同一視** して重複カウントしない(4e メタ「§3.2(同一ブロックの訳文 — 原文ヒットと同一視)」)。クエリ翻訳による相互検索は M3。
- セマンティック検索・「似た手持ち論文」は M3。まず確実な全文検索を出す。

## 10. エクスポート(P5)

設定画面(4f)の「エクスポート」カテゴリと同じ 3 形式:

- **論文単位 Markdown(Obsidian 互換)**: メモ+注釈+チャット履歴+リソース一覧(種類・タイトル・URL・ひとことメモ。[12-resources.md](12-resources.md))を 1 つの Markdown に(原文引用・アンカー情報つき)。ビューアの情報パネルからは注釈 Markdown / 原文 PDF も個別に出せる([04-viewer.md](04-viewer.md))。
- **BibTeX / CSV**: 書誌+ステータス+タグ+日付のライブラリ単位エクスポート。
- **JSON 一括**: 全量エクスポート。
- Obsidian 等への連携は Markdown の体裁を整えることで間接対応(専用プラグインは非目標)。

## 11. 受け入れ基準

- [ ] ステータス 6 値(読む予定/すぐ読む/読んでいる/読んだ/あとで再読/保留)が仕様の色ドットで一貫表示される
- [ ] クイックフィルタ 5 種の件数合計がライブラリ総数と一致する(未読+途中+読了+要再確認=すべて)
- [ ] ステータスが勝手に変わることがない(既定は提案のみ。設定で自動適用/提案する/提案しないを切替可能)
- [ ] 読了フローの全項目を「すべてスキップ」で入力なしに完了でき、読了日・累計読書時間は自動記録される
- [ ] テーブルビューが 10 列固定で表示され、複数選択でフローティング一括操作バー(ステータス変更/タグ追加/コレクションへ)が出る
- [ ] 「この条件を保存」で保存したフィルタがサイドバーに件数付きで表示される
- [ ] キューが 6 本になると「積みすぎかも?」バナーが表示され、閉じることができる
- [ ] コレクションで順序ドラッグ・締切・担当者・発表時間・進捗集計が機能し、並び順が共有ページに反映される
- [ ] コレクション共有リンクをアカウントなしで閲覧でき、noindex が設定され、許可したメモのみ表示される
- [ ] 通知 3 種(翻訳完了/ステータス提案/締切リマインド)が届き、「すべて既読にする」で既読化できる
- [ ] 「EMA teacher」のような英語クエリで訳文側の対応ブロックにもヒットし(クロス検索・同一視)、源別バッジ付きで該当位置へ飛べる(S5)
- [ ] BibTeX エクスポートが主要リファレンスマネージャで読み込める
