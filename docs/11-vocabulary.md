# 11. 語彙帳(英語学習)

> 語彙帳は「論文から育てる英語学習」の機能(4d)。読んでいる論文の原文から単語・コロケーション・イディオムを**文脈センテンスごと**保存し、文脈での語義・解釈のしかた・語源・覚えるコツを AI が自動生成、忘却曲線ベースの復習(SRS)で定着させる。**訳語統一のための用語集([03-translation.md](03-translation.md) の3層用語集)とは別物**である。

## 1. 位置づけ — 用語集との違い

語彙帳と用語集は目的も対象もライフサイクルも異なる。混同しない(A6)。

| | 用語集([03-translation.md](03-translation.md)) | 語彙帳(本書・4d) |
|---|---|---|
| 目的 | 翻訳の訳語を論文内・ユーザー内で統一する | ユーザーの英語力(読解語彙)を育てる |
| 対象 | 専門用語・頭字語・手法名 | 一般学術英語(単語/コロケーション/イディオム) |
| 単位 | 原語↔訳語のペア | 語彙+文脈センテンス+学習コンテンツ |
| 使われ方 | 翻訳パイプラインに適用される | ユーザーが読み返し・復習する |
| 画面 | ビューアの用語パネル | 第一級画面「語彙帳」(サイドバーにホーム/ライブラリと並ぶ。件数バッジ付き) |

- 語彙帳はユーザー単位で、**全論文を横断する 1 冊**。エントリは出典論文([01-domain-model.md](01-domain-model.md) の VocabEntry、LibraryItem に従属)と位置アンカーで紐づく。
- 画面サブタイトルは「46 語 — 読んだ論文の文脈から」(4d)。辞書アプリではなく「自分が読んだ文脈」から育てることが価値の中心。

## 2. 登録フロー: ビューアから文脈ごと保存

登録の入口はビューアの選択メニューのみ(1b)。語彙帳画面から手入力で追加する UI は持たない(4d のヒント文言「本文で選択 → 「語彙に追加」で文脈ごと保存されます」)。

1. ビューアで**原文テキストを選択**する(原文モード/対訳モード、または訳文モードの対訳ポップ内の英語原文。1b では対訳ポップ内の一文選択として描かれている)。
2. 選択メニュー(4色ハイライト / コメント / ✦ AIに質問 / **語彙に追加** / コピー)から「語彙に追加」を選ぶ([04-viewer.md](04-viewer.md))。
3. 以下が**自動で紐づいて**保存される。ユーザーの追加入力は不要(1アクション、P2):
   - **文脈センテンス**: 選択範囲を含む原文センテンス全体+選択範囲のハイライト位置
   - **出典**: 論文タイトル+セクション参照(例「Rectified Flow · §2.1」)+位置アンカー([01-domain-model.md](01-domain-model.md) の共通位置参照。「原文で見る →」ジャンプに使う)
   - **追加日時**
4. 保存直後、学習コンテンツ(§4)がバックグラウンドで自動生成される。

- **決定**: 保存時の確認は控えめなトースト(「語彙帳に追加しました」)のみとし、生成完了を待たせない。理由: 読書の中断を最小にする(P2/P6)。生成完了前に語彙帳を開いた場合は該当フィールドに生成中表示を出す。
- **決定**: 生成に失敗したエントリは語彙本体+文脈センテンス+出典だけで保存し、詳細パネルに失敗理由と「生成を再試行」を表示する。黙って捨てない(P3)。
- **決定**: 同一見出し語が既に語彙帳にある場合は重複エントリを作らず、「既に語彙帳にあります」トーストから既存エントリを開く。理由: 一覧とSRSの重複を防ぐ。複数文脈の蓄積は将来検討。

## 3. 語彙種別(3分類)

保存時に AI が種別を自動判定する(編集可)。一覧・フィルタ・バッジで常時明示する(4d)。

| 種別 | 例(4d 逐語) | バッジ色 |
|---|---|---|
| 単語 | 「circumvent」「albeit」「amortize」「de facto」 | グレー(背景 #F1EFE9 / 文字 #777B81) |
| コロケーション | 「in tandem with」「hinge on」 | 青系(背景 rgba(88,132,170,0.16) / 文字 #4A6E8E) |
| イディオム | 「boil down to」「shed light on」 | 紫系(背景 rgba(110,90,126,0.14) / 文字 #6E5A7E) |

- 種別とは別に、細かい分類ラベル(品詞等。例「句動詞」)を持ち、詳細パネルのメタ行に表示する。

## 4. AI 自動生成コンテンツ

保存時に**文脈センテンスを入力として**以下を自動生成する。すべてユーザー編集可能で、詳細パネルに「AI生成 · 編集可」バッジを表示する(4d)。

| フィールド | 内容(4d の例) |
|---|---|
| 発音記号(IPA) | 「/ˌbɔɪl ˈdaʊn tə/」 |
| 種別・分類ラベル | イディオム / 「句動詞」 |
| 文脈での語義 | 短形(一覧用: 「要するに〜に帰着する」)+長形(詳細用: 「(複雑なものが)煮詰まって結局〜に帰着する。この文では「学習目的は結局、単純な最小二乗回帰になる」。」)。**辞書義の羅列ではなく、この文でどういう意味かを説明する** |
| 解釈のしかた | 読み方の型の解説(例「解釈のしかた(句動詞の読み方)」: boil+down+to の物理イメージ分解)。次に似た表現に出会ったとき自力で読めるようにする |
| 語源メモ | 「boil ← ラテン語 bullīre(泡立つ)。bubble、ebullient(沸き立つ→熱狂的な)と同族。」 |
| ✦ 覚えるコツ | 記憶フック(例「カレーを煮詰めるイメージ。…」)。琥珀系の強調カードで表示 |
| よく出る形・近い表現 | 頻出パターンと類義表現(例「it boils down to whether… / come down to(ほぼ同義)/ amount to(積み上がって〜になる)」) |

- 生成物は AI 生成であることを常に明示し(P1)、編集したフィールドは以後の再生成で上書きしない(手動編集の保護は [03-translation.md](03-translation.md) の翻訳ブロックと同方針)。

## 5. 語彙帳画面(一覧 UI)

マスター/ディテール構成: 左=語彙リスト、右=詳細パネル(幅 400px)。行クリックで右に詳細を表示し、選択行をハイライトする(4d)。

### 5.1 見出し行・フィルタ

- タイトル「語彙帳」+件数サブ「46 語 — 読んだ論文の文脈から」。サイドバーの語彙帳件数と同期。
- **語彙検索ボックス**「語彙を検索」: 語彙帳内のインクリメンタル検索。グローバル検索(⌘K、[06-library.md](06-library.md) の横断検索)とは**別**の専用入力。
- プライマリボタン「**復習をはじめる**」+復習待ち件数バッジ(例「12」)。→ §7
- フィルタチップ: 「すべて 46」「単語 28」「コロケーション 12」「イディオム 6」+「**復習期 12**」。復習期チップは琥珀系(枠 #E4CFA6 / 文字 #8A6A24 / 背景 #FFF9F0)の常時強調表示で、種別チップの選択状態とは別トーン。
- チップ行右端に登録導線ヒント「本文で選択 → 「語彙に追加」で文脈ごと保存されます」(空状態に頼らず常時表示)。

### 5.2 リスト

| 列 | 内容 | 備考 |
|---|---|---|
| 語彙 | 見出し語(Source Serif 4)+種別バッジ | ソート可(ヘッダ「語彙 ↑」= 昇順表示中) |
| 文脈での語義 | 語義の短形 | |
| 出典 | 論文名+§(例「Rectified Flow · §2.1」) | |
| 追加 | 相対/絶対日時(「昨日」「3日前」「6/28」) | |

- リストフッタに常設注記: 「語義・語源・コツは保存時に文脈から自動生成されます(編集可)· 復習は忘却曲線に沿って出題」。
- **決定**: 既定ソートは追加日の降順(新しい語が上)。理由: デザインは語彙昇順の状態を示すのみで既定を規定しないが、直近に読んだ語から復習・確認する使い方が主のため。列ヘッダクリックで語彙/追加のソート切替。

## 6. 詳細パネル

### 6.1 ヘッダ

- 見出し語(例「boil down to」)+発音記号(IPA、例「/ˌbɔɪl ˈdaʊn tə/」)+種別バッジ(例「イディオム」)
- メタ行: 分類ラベル+追加元(例「句動詞 · Rectified Flow §2.1 で追加 · 」)+バッジ「**AI生成 · 編集可**」

### 6.2 本文 6 セクション(上から順)

1. **文脈での語義**(長形。キーとなる訳を太字強調)
2. **文脈センテンス**: 原文英語(Source Serif 4・イタリック・左罫線の引用体裁)。対象語を琥珀色マーカーでハイライト。末尾に「 — §2.1 · **原文で見る →**」— クリックでビューアの該当センテンス位置へジャンプ(位置アンカー経由。[04-viewer.md](04-viewer.md))
3. **解釈のしかた**(薄ベージュカード)
4. **語源メモ**
5. **✦ 覚えるコツ**(琥珀系カード。見出し色 #8A6A24)
6. **よく出る形・近い表現**

### 6.3 フッタ(復習ステータス)

- 次回復習の表示: 「次の復習: 明日(2 回目)」
- 自己評価 2 ボタン: セカンダリ「**まだあやしい**」/ プライマリ「**✓ 覚えた**」→ §7 の規則でスケジュール更新
- **決定**: 詳細パネルのオーバーフローメニューからエントリの削除と各フィールドの再生成を行える。理由: デザインに削除 UI は描かれていないが、ユーザーデータの基本操作として必須(誤登録の回収)。削除は取り消し可能なトーストを伴う(P3)。

## 7. 復習(SRS)

忘却曲線に沿った間隔反復(4d フッタ注記「復習は忘却曲線に沿って出題」)。

### 7.1 スケジュール規則

**決定**: SM-2 系を簡略化した固定段階方式とする。理由: デザインが示すのは「次の復習: 明日(2 回目)」と 2 択評価のみで、可変難易度係数(EF)を持つ完全な SM-2 は 2 択 UI と釣り合わず、挙動の説明可能性(P3: 勝手に変えない・予測可能)を優先するため。

| 段階 | 1 | 2 | 3 | 4 | 5 |
|---|---|---|---|---|---|
| 次回までの間隔 | 1日 | 3日 | 7日 | 14日 | 30日 |

- 保存時: 段階 1、次回復習 = 翌日(4d の例「次の復習: 明日(2 回目)」= 昨日追加した語の 1 回目復習を今日「まだあやしい」で終え、翌日に 2 回目が来る状態)。
- 「✓ 覚えた」: 段階を 1 つ進め、次回 = 今日+新段階の間隔。
- 「まだあやしい」: **段階 1 にリセット**、次回 = 翌日(同一セッション内では後回しにして再出題)。
- 段階 5 を「✓ 覚えた」で通過した語は**習得済み**とし、復習キューから外す(一覧には残る。「まだあやしい」相当の操作でいつでも段階 1 に戻せる)。理由: 復習キューの無限肥大を防ぐ(P6: 管理を強要しない)。
- **復習期(due)**: 次回復習日 ≤ 今日 のエントリ。件数が「復習期」チップと「復習をはじめる」の件数バッジに出る。

### 7.2 復習セッション

- 「復習をはじめる」で復習期の語を対象にセッションを開始する。
- **決定**: 出題はフラッシュカード形式 — まず見出し語+文脈センテンス(語義は伏せる)を提示 → めくって語義・コツを表示 → 「まだあやしい」/「✓ 覚えた」で評価して次へ。理由: デザインは開始ボタンと評価 2 択のみを規定しており、最小の出題形式として詳細パネルの既存構成を再利用する。新しい出題形式(穴埋め・音声等)は発明しない。
- セッションは途中でやめてよい(評価済みの語だけ更新)。未評価の語のスケジュールは変えない(P3)。
- 復習の催促通知・連続記録はしない(P6)。復習期の可視化は語彙帳画面内のチップ/バッジに留める。

## 8. 生成に使う LLM

- 語彙コンテンツ生成は**小型・低価格帯モデル**を既定とする(短文生成でレイテンシ・コスト重視。モデル階層と既定ルーティングは [09-nonfunctional.md](09-nonfunctional.md))。
- プロバイダは OpenAI / Anthropic / Google / DeepSeek から用途別に切替可能(運営既定+ユーザー設定+BYOK。A9)。
- 出力は構造化(§4 のフィールド構成)で受け取り、フィールド単位で保存・編集する。

## 9. エクスポート(P5)

- **Markdown エクスポート**: 語彙帳全体または現在のフィルタ結果を Markdown で書き出す(見出し語・種別・語義・文脈センテンス・出典・生成コンテンツを含む)。Obsidian 互換の体裁は [06-library.md](06-library.md) のエクスポート方針に従う。
- 全量 JSON 一括エクスポートに VocabEntry(SRS 状態含む)を含める。
- **Anki エクスポート**(TSV 形式、外部依存なし): 語彙帳を Anki 取り込み用の TSV として書き出す。**実装済み**([10-roadmap.md](10-roadmap.md))。

## 10. 受け入れ基準

- [ ] ビューアで原文を選択し「語彙に追加」1 アクションで、文脈センテンス・出典(論文+§)・追加日が自動で紐づいて保存される
- [ ] 保存後に語義(短形/長形)・IPA・種別・解釈のしかた・語源・覚えるコツ・よく出る形が自動生成され、すべて編集できる。編集したフィールドは再生成で上書きされない
- [ ] 生成に失敗しても語彙+文脈+出典は保存され、失敗理由と再試行導線が表示される(黙って消えない)
- [ ] 一覧が種別チップ(単語/コロケーション/イディオム)と「復習期」チップで絞り込め、語彙列・追加列でソートできる
- [ ] 「語彙を検索」が語彙帳内のみを対象とし、グローバル検索(⌘K)と混ざらない
- [ ] 詳細パネルの「原文で見る →」で出典論文の該当センテンス位置がハイライトされた状態でビューアが開く
- [ ] 「✓ 覚えた」/「まだあやしい」で §7.1 の規則どおり次回復習日が更新され、「次の復習: 明日(2 回目)」形式で表示される
- [ ] 復習期件数が「復習期」チップと「復習をはじめる」ボタンのバッジで一致する(サイドバーの語彙帳バッジは**総語数**を表示し、§5.1 の件数サブと同期する)
- [ ] 語彙帳を Markdown でエクスポートでき、文脈センテンスと出典が含まれる
- [ ] 語彙帳を Anki テキストインポート形式(TSV)でエクスポートできる(`GET /api/vocab/export/anki`)
- [ ] 用語集([03-translation.md](03-translation.md))への操作が語彙帳に影響しない(独立した機能である)
