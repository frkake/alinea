# 08. ブラウザ拡張機能(Chrome / Edge)(3a)

> 拡張の仕事は「見つけた論文を 2 秒でライブラリに入れる」こと。**拡張は本プロダクト唯一の取り込み経路**であり(アプリ内「+追加」ボタン/取り込みダイアログは持たない)、読む機能はサイト側に寄せて拡張は薄く保つ。「ツールバー → 保存」の 2 クリックでライブラリへ。

## 1. 基本方針

- **取り込みの一本化**: 論文の取り込み(URL 送信・PDF 直接送信)はすべてこの拡張から行う。Web アプリ側には追加ボタン・アップロード UI を置かない([02-ingest.md](02-ingest.md))。
  - 理由: 「論文に出会う場所(ブラウザ)で 1 アクション」という P2 を徹底し、取り込み UI の二重管理を避けるため(確定デザインの設計判断)。
- Manifest V3。単一コードベースで Chrome / Edge 両対応(Chromium 系)。
- 権限は最小: `activeTab` + 拡張ポップアップを基本とし、**全サイト常駐のコンテンツスクリプトは持たない**(プライバシーと審査の観点。arXiv ドメイン限定のページ内ボタンのみオプトイン設定で提供。§5)。
- 原則 **URL のみを送信** し、取得・解析はサーバーで行う(§4)。ポップアップにも「URL のみを送信します — 取得・解析はサーバーで実行」と常時明記する。
- 認証: サイトとのセッション共有(cookie)を第一候補、不可なら拡張専用トークン。未ログイン時はポップアップからログイン導線。

## 2. ポップアップの 4 状態

ツールバーの「訳」アイコンをクリックすると、現在タブを判定して 1 つのポップアップ(幅 372px)で完結する。状態は次の 4 つ。

| # | 状態 | 発火条件 | ヘッダ表示 |
|---|---|---|---|
| 1 | 保存前(メイン) | arXiv ページを表示中・未保存 | 「訳読に保存」+緑バッジ「arXiv 論文を検出」 |
| 2 | 保存直後 | 状態 1 で保存を実行 | 「保存しました」+✓ 成功バッジ |
| 3 | 既にライブラリ | 保存済み論文のページを表示中 | 「既にライブラリにあります」 |
| 4 | 一般ページ PDF | arXiv 以外で PDF を表示中 | 「訳読に保存」+グレーバッジ「PDF を表示中」 |

各状態のヘッダ右端に「⚙」(拡張設定)を置く。

### 2.1 状態 1: 保存前(arXiv 検出)

arXiv の論文ページを自動検出し、保存前に以下を 1 画面で提示する。

1. **書誌プレビュー**(自動抽出): タイトル / 著者短縮表記・会議・ID(例: 「Liu, Gong, Liu · ICLR 2023 · arXiv:2209.03003 v3」)。
2. **品質見込み**: LaTeX ソースの有無を保存前に判定し、緑文字で「✓ LaTeX ソースあり — 品質レベル A 見込み」と予告する(品質レベル A/B は [02-ingest.md](02-ingest.md) 参照)。
   - 理由: 保存後に品質が判明するのでは遅い。翻訳品質の期待値を保存前に示すことで P3(黙って壊れない)を取り込み時点から満たす。
3. **ステータス 3 択**(単一選択ピル): 「✓ 読む予定」(既定・選択中)/「すぐ読む」/「読んでいる」。選択中はアクセント枠+淡アクセント面+✓。
   - 決定: 拡張では全 6 ステータスではなく取り込み時に意味のある 3 つのみ提示する(デザイン準拠)。「読んだ」等への変更はサイト側またはポップアップ状態 3 の「ステータス変更 ▾」で行う。
4. **タグ**: 既存チップ(例: 「flow ×」、× で削除)+自由入力「追加…」+システム提案のワンクリック追加「提案: distillation +」(AI/共起提案)。
5. **追加先コレクション**: ドロップダウン(例: 「輪読会 2026-07 ▾」)。
6. **ひとことメモ**: 任意の短文入力(プレースホルダ「ひとことメモ…」)。
7. **保存ボタン**(プライマリ): 「保存 ⏎」。クリックまたは **Enter キー**で実行 → 同ポップアップが状態 2 に遷移。
8. プライバシー注記: 「URL のみを送信します — 取得・解析はサーバーで実行」。

### 2.2 状態 2: 保存直後

同一ポップアップ内で結果+進捗表示に切り替わる(別画面に飛ばさない)。

- ヘッダ「保存しました」+✓。
- 論文カード: サムネイル+タイトル+**パイプライン進捗行**「✓ 書誌」→「✓ 構造化」→「翻訳中 12%」+プログレスバー。段階と進捗率の意味は [02-ingest.md](02-ingest.md) のパイプライン定義に従う。
- ボタン: プライマリ「サイトで開く ↗」(訳読 Web アプリの該当論文ビューアを新規タブで開く。処理途中でも読める部分から開ける=段階的提示 P2)/セカンダリ「閉じる」。
- 説明文: 「進捗はツールバーのバッジにも表示されます(処理中スピナー → 完了チェック)。読める部分から開けます。」

### 2.3 状態 3: 既にライブラリにある論文(重複を作らない)

保存済み論文のページでは重複保存 UI を出さず、既存レコードを表示する。

- 現在ステータスのドット付きピル(例: 「読んでいる」)+「2026/07/02 追加 · 進捗 42%」。
- 前回読了位置: 「前回: §2.1 整流フロー · 昨日 21:52」(節番号+節タイトル+時刻)。
- ボタン: プライマリ「続きから開く ↗」(前回位置からビューアを開く。[04-viewer.md](04-viewer.md))/セカンダリ「ステータス変更 ▾」(ドロップダウンでステータスをその場で変更)。

### 2.4 状態 4: 一般ページで PDF 表示中(明示操作でのみ送信)

arXiv 以外の一般ページで PDF を表示している場合。

- 書誌プレビュー: タイトル+「書誌は推定」バッジ(ローカル推定であることを明示)+ URL 表示。
- **警告ボックス**(琥珀): 「このページはサーバーから取得できない可能性があります(学内ネットワーク等)。ボタンを押したときだけ、このタブの PDF を直接送信します — 自動送信はしません。」
- プライマリボタン: 「このタブの PDF を送信」。**ユーザーの明示クリック時のみ**タブ内 PDF バイト列をアップロードする。
- 注記: 「private 論文として保存され、共有されません」。

## 3. ポップアップ共通要素

### 3.1 直近の取り込み(フッタ)

ポップアップ下部に直近の取り込み履歴を **3 件** 表示する(旧仕様の 5 件からデザイン準拠で変更)。確定デザインでフッタが確認できるのは状態 1(保存前)のみ。
  - 決定: 状態 2〜4 でも同フッタを表示する(デザインの右列モックは省略とみなす。処理中ジョブの視認性を優先)。

- 処理中: スピナー+タイトル+進捗率(例: 「翻訳中 68%」)。
- 完了: 緑 ✓+タイトル+完了時刻(例: 「今日 8:02」「7/01」)。
- 各行はライブラリの該当論文への導線とする。
  - 決定: デザインではリンク先が明示されていないが、履歴行クリックでサイト側の該当論文ビューアを開く(状態 2 の「サイトで開く ↗」と同じ挙動)。失敗した取り込みは失敗表示+サイトへの導線を出す(§6)。

### 3.2 ツールバーアイコンのバッジ

- アイコン右上に **琥珀色ドット(#C49432)** を表示する状態がある = 処理中/未読通知あり。
- 取り込み処理中はスピナー、完了でチェックに変わる(状態 2 の説明文で利用者に告知)。
- 通知の本体(翻訳完了・提案等)はサイト側の通知ポップオーバーで扱う([06-library.md](06-library.md))。拡張のドットはその存在示唆にとどめる。

## 4. 送信プロトコル

- 原則 **URL のみを送信** し、PDF/LaTeX ソースの取得と解析はサーバーで行う(拡張を薄く保つ。ページ内容は送らない)。
- サーバーから到達できないコンテンツ(学内ネットワーク・購読出版社・認証付きページ)の場合のみ、タブ内の PDF バイト列を拡張から直接アップロードする。この動作は毎回ユーザーの明示操作(「このタブの PDF を送信」ボタン)で行い、**自動送信はしない**。
- 拡張から直接送信されたコンテンツは **private 論文** として扱い、共有ページ等に露出しない([02-ingest.md](02-ingest.md), [09-nonfunctional.md](09-nonfunctional.md))。

## 5. arXiv ページ内「訳 保存」ピル(オプトイン)

設定で有効化した場合のみ、arxiv.org の論文ページ(abs)のタイトル横に「訳 保存」ピルボタン(ミニロゴ+「保存」)を注入する。**既定はオフ**。コンテンツスクリプトは arXiv ドメイン限定。

- クリックでワンクリック保存(状態 1 の既定値で保存)。
- 保存後はピルが「✓ 保存済み」表示に変わる。
- 理由: 常駐スクリプトを持たない方針(§1)と、arXiv を高頻度に回遊するユーザーの 1 クリック化要望の両立。オプトイン+ドメイン限定で権限を最小に保つ。

## 6. エラーとオフライン

- 送信失敗時は拡張内キューに保持し、再試行ボタンを表示する(黙って失われない。P3)。ブラウザ再起動後もキューは残る。
- サイト側の取り込み失敗(パイプライン失敗)はポップアップの「直近の取り込み」欄に失敗表示+サイトへの導線を出す。

## 7. 非目標

- 拡張内での翻訳表示・チャット(サイトへ誘導する)
- 閲覧履歴の収集、開いているタブの自動スキャン
- arXiv 以外の一般 HTML ページからの書誌検出・保存(v1 では対象外。取り込み対象は arXiv ページと表示中 PDF の 2 系統)
  - 決定: 旧仕様にあった「一般 HTML ページの DOI/メタタグ検出保存」は確定デザインに存在しないため v1 スコープから外す。対象外ページでポップアップを開いた場合は取り込み不可の案内と「直近の取り込み」のみを表示する。
- Firefox / Safari 対応(将来検討。コードは WebExtension 標準に寄せておく)

## 8. 受け入れ基準

- [ ] アプリ内に取り込み UI が存在せず、拡張が唯一の取り込み経路である
- [ ] arXiv abs ページで、クリック 2 回(ツールバー → 保存)以内にライブラリに入る
- [ ] 保存前に書誌プレビューと「✓ LaTeX ソースあり — 品質レベル A 見込み」判定が表示される
- [ ] 保存前にステータス(読む予定/すぐ読む/読んでいる)・タグ(提案含む)・コレクション・ひとことメモを指定でき、サイト側に即時反映される
- [ ] 保存は Enter キーでも実行できる
- [ ] 保存直後、同一ポップアップ内にパイプライン進捗(書誌✓→構造化✓→翻訳中%)と「サイトで開く ↗」が表示され、処理途中でも読める部分から開ける
- [ ] 既存論文のページでは重複保存 UI が出ず、現ステータス・追加日・進捗・前回位置と「続きから開く ↗」「ステータス変更 ▾」が表示される
- [ ] 一般ページの PDF は警告表示のうえ明示クリック時のみ送信され、private 論文として保存される(自動送信が発生しない)
- [ ] ポップアップ下部に直近の取り込み 3 件(処理中は進捗率、完了は時刻)が表示される
- [ ] ページ内「訳 保存」ピルは既定オフで、有効化時は arXiv 限定で動作し、保存後「✓ 保存済み」に変わる
- [ ] 処理中/通知ありの間、ツールバーアイコンに琥珀ドットが表示される
- [ ] ブラウザ再起動後もキューされた失敗送信が残っている
- [ ] Chrome / Edge の両ストア審査を通過できる権限構成(activeTab 基本)である
