# 09. 非機能要件

> 性能・コスト・信頼性・法務。特にライセンス(§5)は機能仕様(翻訳の共有可否・記事モードの図表転載・公開範囲)を直接規定するため、実装前に必ず読むこと。コスト設計(§3)は LLM / 画像生成のマルチプロバイダ構成(用途別ルーティング+フォールバック+BYOK)を規定する。

## 1. 性能目標(p50 / p95)

| 操作 | p50 | p95 |
|---|---|---|
| ライブラリ・ダッシュボード表示 | 1 秒 | 3 秒 |
| ビューア初期表示(翻訳済み論文) | 2 秒 | 5 秒 |
| 取り込み: 拡張の保存→ライブラリにカード表示 | 3 秒 | 10 秒 |
| 取り込み: アブスト訳+✦3行要約 | 20 秒 | 60 秒 |
| 取り込み: 読み始められる(readable) | 60 秒 | 3 分 |
| 全文翻訳完了(〜10 ページ) | 5 分 | 15 分 |
| チャット初回トークン | 5 秒 | 15 秒 |
| 記事モード初回生成((1h)) | 30 秒 | 90 秒 |
| 解説図(ラスター)1 枚生成 | 20 秒 | 60 秒 |
| 語彙エントリ保存時の AI 生成(語義・語源など) | 3 秒 | 10 秒 |
| 横断検索((4e))・ライブラリ全文検索 | 1 秒 | 3 秒 |

- 長時間処理(取り込み・翻訳・記事生成・図生成)はすべて非同期ジョブ+進捗表示+完了通知(通知アイコン◷(4a))で扱い、ページを閉じても継続する。
- 決定: 記事モード・解説図・語彙生成の目標値は確定デザインに記載がないため本doc で新設した。いずれも「ユーザーが画面の前で待つ操作」であり、目標がないと §3 のモデル選択(速度と品質のトレードオフ)を判断できないため。

## 2. 信頼性

- ジョブは段階ごとに冪等・再開可能(同一入力の再実行が安全)。自動リトライは指数バックオフで 3 回、以後は手動再試行(情報パネルの「再取り込み」「処理ログ」(2a))。
- 部分成功を正の状態として扱う(例: 図抽出だけ失敗しても本文は読める。「付録は未翻訳」のような部分完了はタイムライン文言で明示する。P3)。
- 原データ(SourceAsset)は保持し、パーサ改良時の再処理・品質レベル B→A への昇格を可能にする([02-ingest.md](02-ingest.md))。品質レベルは **A(LaTeX から完全構造化)/ B(PDF 由来)** の 2 段階で、バッジで常時明示する。
- LLM / 画像生成プロバイダの障害時: 翻訳・生成ジョブは §3.5 のフォールバック連鎖で代替プロバイダに切り替え、全滅時は待機キューに退避。チャットはエラーを明示する(サイレント劣化しない。P3)。

## 3. コスト設計

### 3.1 コスト制御原則

最大の変動費は LLM・画像生成 API。制御手段は仕様に組み込み済み:

- 共有翻訳キャッシュ([03-translation.md](03-translation.md))— 人気論文ほど限界費用ゼロに近づく
- 既定で訳さない領域(付録・参考文献・表セル。付録は「開くと翻訳します(オンデマンド)」)
- オンデマンド生成: 直訳スタイル・記事モード・概要図・解説図は自動生成しない(ユーザー操作を起点にする)
- 長大論文(30 ページ超)のセクション選択提案((4f))
- プロバイダのプロンプトキャッシュ活用(翻訳の共有システムプロンプトはキャッシュ対象に設計する)

### 3.2 プロバイダ構成と用途別ルーティング

- **テキスト生成**(翻訳/チャット/要約/記事/語彙)は **OpenAI / Anthropic / Google / DeepSeek** の 4 プロバイダを、**画像生成**(解説図)は **OpenAI / Google / xAI** の 3 プロバイダを切替可能にする。
- ルーティングは**用途別**(translation / chat / summary / article / figure / vocab)に設定テーブルで管理する。適用順: 運営既定 → ユーザー設定((4f)) → BYOK。
- **モデル ID は設定で変更可能**とする(ハードコードしない)。モデルの世代交代が速く、価格改定・旧 ID 廃止(例: DeepSeek の `deepseek-chat`/`deepseek-reasoner` は 2026-07-24 廃止)に運用で追随する必要があるため。
- プロバイダ抽象化層が SDK 差異(structured outputs・ストリーミング・温度パラメータ不可等の制約)を吸収する(§7)。

### 3.4 既定ルーティング案

用途ごとの既定(運営キー使用時)。**すべて設定((4f))で変更可能**。

### 3.5 フォールバック連鎖・BYOK・クォータ

- 各用途に**フォールバック連鎖**(第 1 → 第 2 → …)を構成として持つ。切替はレート制限・障害・タイムアウトを契機とし、ユーザーには処理ログで判別可能にする(黙って品質を落とさない。P3)。
- **BYOK**: ユーザーは自分の API キーを設定でき、その場合クォータを消費しない。キーの保管要件は §4。
- ユーザー別の**月次クォータ**(全文翻訳本数・チャットメッセージ数・画像生成枚数)を計測・表示する。超過時の扱い(待機 or BYOK or 課金)は [00-product.md](00-product.md) の未決事項の決定に従う。
- クォータ残量は設定画面に常時表示し、突然の利用停止で驚かせない。

## 4. セキュリティ・プライバシー

- ライブラリ・注釈・メモ・チャット・語彙・記事モード・読書統計は **既定で完全非公開**。v1 で公開されるのは**コレクション共有ページ((4c))のみ**: アカウント不要・閲覧専用・noindex で、内容は書誌+✦要約+「共有ページにメモを含める」で許可したメモに限る([06-library.md](06-library.md))。記事の公開・限定公開は v2 検討([07-figures-and-articles.md](07-figures-and-articles.md))。
- 共有リンクトークン(`yakudoku.app/c/{token}`)は十分なエントロピーを持ち、発行状況の確認と無効化ができる。
- アップロード PDF・拡張から送信されたコンテンツは所有者のみアクセス可(private)。サムネイル等の派生物も同様。
- ユーザーコンテンツ(メモ・チャット)を LLM の学習に使わない。外部 LLM / 画像生成 API に送るデータの範囲(論文本文・質問・メモ)を、送信先プロバイダが複数あることを含めてプライバシーポリシーに明記する。
- 決定: BYOK の API キーは保存時に暗号化し、画面上はマスク表示・再表示不可(再入力のみ)とする。デザインには描かれていないが、外部課金に直結する秘匿情報のため要件として明記する。
- 認証: OAuth(Google / GitHub)+ メール。セッションは標準的な Web セキュリティ要件(CSRF / XSS 対策)を満たす。
- 読書時間の計測はユーザーに可視・無効化可能(「読書時間を記録しています(設定でオフにできます)」(2a))。
- アカウント削除で個人データを完全削除([01-domain-model.md](01-domain-model.md))。

## 5. 著作権・ライセンス(機能を規定する制約)

### 5.1 基本整理

- 論文の翻訳は翻案にあたるため、**全文翻訳の公開はライセンスが許す場合を除き不可**。本サービスの全文翻訳は「取り込んだ本人が読むための私的な表示」に限定する(P7)。
- v1 の公開面はコレクション共有ページ((4c))のみで、原文・訳文の本文は一切含まない(書誌+要約+許可メモ)。**記事(自らの解説という二次的創作)と概要図(自作図)の公開は v2 の検討事項**であり、その際も本節のマトリクスに従う。
- 記事モード((1h))は v1 では本人のみが閲覧する表示モードだが、**図表転載の可否判定・ライセンスバッジ・クレジット自動付記は v1 から実装する**(「CC BY 4.0 — 図表転載可」「記事への図表埋め込み時、クレジットを自動付記します。」(2a))。理由: 私的表示の段階から権利表示を正しく行い、v2 の公開解禁時に既存記事を作り直させないため。
- 共有翻訳キャッシュ([03-translation.md](03-translation.md))はサーバー内部のコスト最適化であり、ユーザー体験上は各自の私的翻訳として提示する。訳文自体を公開・配布する機能は提供しない。

### 5.2 arXiv ライセンス別マトリクス

Paper に記録したライセンス([02-ingest.md](02-ingest.md))で以下を機械判定する:

| ライセンス | 本人向け翻訳表示 | 記事モードへの図表転載(v1 は私的表示、v2 公開時も同判定) | 備考 |
|---|---|---|---|
| CC BY 4.0 | ○ | ○(クレジット自動付記) | |
| CC BY-SA 4.0 | ○ | ○(クレジット+記事の該当部分に SA 表示) | |
| CC BY-NC(-SA) 4.0 | ○ | ○(非商用の範囲。サービスが有料化する場合は再検討) | |
| CC BY-ND 4.0 | ○ | △(改変不可: 図はそのまま、キャプション翻訳は図と分離して表示) | |
| CC0 | ○ | ○ | |
| arXiv 非独占ライセンス(既定) | ○(私的表示) | **×**(リンクカード・自作概要図で代替、[07-figures-and-articles.md](07-figures-and-articles.md)) | 多数派。ここを既定挙動として設計する |
| 不明 / 出版社 PDF / アップロード PDF | ○(私的表示) | × | 該当論文はコレクション共有ページでも書誌のみに縮退 |

### 5.3 運用

- 権利者からの削除要請の受付窓口と手順を用意する(v1 の対象は共有ページ上の書誌・要約。v2 で記事公開を解禁した場合は公開記事の図表を含む)。
- 記事モードの出典ブロックは自動挿入・削除不可((1h))。v2 の公開機能でも強制を維持する。
- arXiv へのアクセスは API 利用規約・レート制限(バースト禁止・適切な User-Agent)を遵守し、取得物(LaTeX ソース・PDF)をキャッシュして再取得を減らす。

## 6. アクセシビリティ・対応環境

- WCAG 2.1 AA を目標(コントラスト・キーボード操作・数式代替テキスト)。
- デスクトップ基準ビューポートは **1440×900**(確定デザインの画面フレーム)。ダークモードを提供する((1c))。
- 対応ブラウザ: Chrome / Edge / Safari / Firefox の最新 2 バージョン。ブラウザ拡張は Chrome / Edge([08-extension.md](08-extension.md))。
- モバイル(iOS Safari / Android Chrome)は**閲覧とステータス変更のみ**。取り込みは拡張機能に一本化されているためモバイルからの取り込みは v1 スコープ外([00-product.md](00-product.md))。
- UI 言語は日本語のみ(v1。Q5)。

## 7. 実装ノート(仕様ではなく制約)

### 7.1 設定画面(4f)の構成

設定は 8 カテゴリの左ナビで構成される(確定デザイン 4f)。各カテゴリの内容仕様は担当文書に分散するため、対応をここに集約する:

| カテゴリ | 主な内容 | 仕様の所在 |
|---|---|---|
| アカウント | 認証(OAuth/メール)、BYOK API キー(暗号化保存・マスク表示) | 本書 §3.5・§4 |
| 表示 | テーマ(ライト/ダーク/システム)、アクセント 4 色、本文書体(明朝⇄ゴシック)、フォントサイズ・行間・本文幅 | [04-viewer.md](04-viewer.md) |
| 翻訳 | 既定スタイル(自然訳/直訳)、付録・表セルの翻訳既定、30ページ超のセクション選択提案、翻訳モデル選択(§3.2) | [03-translation.md](03-translation.md)、本書 §3 |
| 読書の計測と提案 | 読書時間計測 ON/OFF、ステータス自動遷移(自動適用/提案する(既定)/提案しない) | [04-viewer.md](04-viewer.md)、[06-library.md](06-library.md) |
| チャット | 文脈への注釈・メモ包含 ON/OFF、チャットモデル選択(§3.2) | [05-chat.md](05-chat.md)、本書 §3 |
| 通知 | 通知 3 種の個別 ON/OFF | [06-library.md](06-library.md) |
| エクスポート | 論文単位 Markdown / BibTeX・CSV / JSON 一括 | [06-library.md](06-library.md) |
| ブラウザ拡張 | arXiv ページ内ボタン(オプトイン)等 | [08-extension.md](08-extension.md) |

### 7.2 技術制約

技術選定は実装フェーズに委ねるが、仕様が要求する制約のみ列挙する:

- **デザイントークンは確定デザイン(`論文読解システム デザイン.dc.html`)の値を正とする**。出典はデザインのヘッダコメントおよび Tweaks スクリプト。具体的には: アクセント 4 色(#3E5C76 / #4A6B57 / #6E5A7E / #7A5C48)とダーク対応色マップ、`--pr-a` / `--pr-as` / `--pr-am` / `--pr-ad` 系の導出規則(rgba 透過率を含む)、注釈 4 色、ステータス 6 色、書体 4 系統(IBM Plex Sans JP / Noto Serif JP(設定でゴシック切替)/ Source Serif 4 / IBM Plex Mono)と Google Fonts 読み込み指定。実装はトークンを単一ソース(共有パッケージ等)で管理し、デザインとの差分を作らない。
- 数式描画は KaTeX 相当で、サーバーサイドレンダリング可能であること(共有ページ・記事モードの初期表示)。
- 図は二本立て: **全体概要図は構造化図データからの SVG 決定的レンダリング**(同一データ→100%同一デザイン。版管理と SVG ダウンロード(1h) を成立させるため)。**解説図はラスター画像生成 API**(プロバイダ切替可、§3)。ラスター図は画像内テキストの破綻リスクがあるため、重要なテキストは画像内に埋め込まずキャプション側に持つ生成プロンプト仕様とする。
- LLM / 画像生成はプロバイダ抽象化層を介して呼び出し、モデル ID・ルーティング・フォールバック連鎖を設定データとして持つこと(コードに固定しない。§3.2)。
- 全文検索は日本語形態素解析と英語ステミングの両対応で、横断検索の日英クロス(日本語→訳文、英語→原文)とヒット源の区別(本文/訳文/メモ/チャット/記事)を実現できること((4e))。
- ジョブ基盤は段階再開・優先度繰り上げ(オンデマンド先行)をサポートすること。
- オブジェクトストレージに SourceAsset・図アセット(SVG 原データ・生成ラスター)を保持すること(原本再処理可能性・概要図の版管理)。

## 8. 受け入れ基準チェックリスト

- [ ] §1 の性能目標を計測できるテレメトリがあり、p50 / p95 がダッシュボード等で確認できる
- [ ] 取り込み・翻訳ジョブを途中で強制終了→再実行しても二重処理・データ破損が起きない(冪等・段階再開)
- [ ] 用途別ルーティング(翻訳/チャット/要約/記事/語彙/解説図)が設定テーブルで管理され、モデル ID を再デプロイなしで変更できる
- [ ] 第 1 プロバイダを停止させるとフォールバック連鎖で処理が継続し、処理ログから使用モデルが判別できる
- [ ] BYOK キー設定時はクォータを消費せず、キーは暗号化保存・マスク表示される
- [ ] クォータ残量が設定画面に表示され、超過時に定義された挙動(待機 / BYOK 誘導)になる
- [ ] 未ログインで共有ページ URL を開くと書誌+要約+許可メモのみ閲覧でき、noindex が付与されている。それ以外のページは認証必須
- [ ] arXiv 非独占ライセンスの論文で、記事モードに図の直接転載が行われず代替(リンクカード等)になる。CC BY 4.0 では転載+クレジット自動付記される
- [ ] 実装の CSS トークン値が確定デザインのヘッダコメント/Tweaks スクリプトの値と一致する(アクセント導出・注釈色・書体)
- [ ] 同一の概要図データから常にバイト同一の SVG が生成される(決定的レンダリング)
- [ ] 横断検索で日本語クエリが訳文に、英語クエリが原文にヒットし、ヒット源バッジが正しく表示される
