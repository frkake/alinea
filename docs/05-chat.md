# 05. 読解チャット

> チャットの価値は「答えること」ではなく「原文のどこを読めばよいかまで連れて行くこと」。根拠のない回答は本文理解を汚染するため、根拠表示を必須要件とする(P1)。AI生成物は「AI生成」「論文外の知識」「推測」のラベルで本文由来と常に区別する。

## 1. 画面構成(1a)

チャットはビューアのサイドパネル(排他6タブ: チャット / メモ / 注釈 / 図表 / リソース / 情報)の1タブとして常駐する([04-viewer.md](04-viewer.md))。パネル幅 340px。上から:

| 領域 | 内容 |
|---|---|
| タブ行 | 「チャット」選択中はアクセント色+下端2pxインジケータ |
| スレッド/コンテキスト行 | 左「スレッド: メイン ▾」(スレッド切替ドロップダウン)/ 右にスコープチップ「コンテキスト: この論文」 |
| メッセージ領域 | 日時セパレータ(例「今日 21:47」)、ユーザーメッセージ(白カード)、アシスタント回答(カードなし)。発話者ラベルは「あなた」/「✦ アシスタント」 |
| 入力エリア | 定型チップ行(§6)+ 入力ボックス(プレースホルダ「この論文について質問…」、送信ボタン「↑」)+ 固定免責文 |

- 固定免責文(逐語・入力エリア下部に常時表示): 「回答は原文を根拠にします。本文にない内容は「論文外の知識」「推測」と表示されます。」

## 2. スレッドモデルとコンテキストスコープ

- 論文(LibraryItem)ごとにチャットを持つ。既定は 1 本の「メイン」スレッド。スレッド切替・新規作成はスレッド行の「スレッド: メイン ▾」ドロップダウンから行う(例: 「数式の確認」「実装メモ」)。控えめな配置とし、乱立を促さない。
- コンテキストスコープは常に「この論文」であり、スコープチップ「コンテキスト: この論文」で UI が明示する。ライブラリ横断のチャットは v1 では提供しない。
- 選択質問(§4)は新規スレッドを作らず、**アクティブなスレッドにアンカー付きメッセージとして積まれる**。読解の文脈は 1 本の流れとして残るほうが後から読み返せる。
- チャット履歴は保存され、横断検索(4e)の対象になる。検索結果ではヒット源「チャット」バッジ(紫)が付き、「スレッドを開く」で該当スレッドへ遷移する([06-library.md](06-library.md))。論文を開き直せば続きから話せる。
- チャット履歴は記事モードの生成素材にもなる([07-figures-and-articles.md](07-figures-and-articles.md))。

## 3. モデルに与える文脈

- 構造化ドキュメント全体(原文)。コンテキスト長を超える場合は「全セクションの要約 + 質問に関連するセクションの全文 + 選択範囲の周辺全文」で構成する。
- 論文メタデータ(タイトル・著者・年・venue)。
- 直近の会話履歴(アクティブスレッド)。
- ユーザーの注釈・メモ(**設定で除外可能**。既定は含める — 「さっき疑問ハイライトした箇所について聞きたい」が自然に通じるように)。この設定は設定画面(4f)の「チャット」カテゴリに置く。
- 訳文ではなく **原文を正とする**(訳文経由の伝言ゲームで精度を落とさない)。回答は日本語。
- 使用モデルはチャット用途のプロバイダ設定(GPT / Claude / Gemini / DeepSeek、運営既定+ユーザー変更+BYOK)に従う([09-nonfunctional.md](09-nonfunctional.md))。

## 4. 選択質問(コンテキスト付き質問)

- ビューアの選択メニュー「✦ AIに質問」から、選択テキストがアンカー付きの引用チップとしてチャット入力欄に入る。複数箇所を積んでから 1 つの質問を送ることもできる(「この 2 つの式の関係は?」)。
- 図・表・数式ブロックにも同じ操作を提供する(ブロック単位アンカー)。数式ブロックの「✦ この式を説明」、図ポップオーバーの「✦ この図を説明」はこのショートカットである([04-viewer.md](04-viewer.md))。図は画像としてモデルに渡す。
- 送信されたユーザーメッセージは白カードで表示され、根拠チップ(例「式(5) · §2.1」)と引用プレビュー(原文の冒頭を引用ブロックで表示)を伴う。
- チャット側の引用チップ・根拠チップをクリックすると、ビューアが該当箇所へスクロールし一時ハイライトする。逆方向のリンクとして、チャットが参照中のブロックは本文側でアクセント色に強調され、浮きバッジ「✦ チャットの根拠 · 式(5)」が付く(チャット⇔本文の双方向リンク)。

## 5. 根拠表示(本機能の中核)

- 回答には、主張を支える原文箇所への **根拠チップ** を文中インラインで付ける。チップはアンカーであり、クリックでビューアの該当位置へジャンプ+一時ハイライト。
- 粒度は**段落(¶)まで**。チップの形式:

| 対象 | 表記例 |
|---|---|
| セクション | `§3.2` |
| 段落 | `§2.1 ¶4` |
| 数式 | `式(5)` |
| 図 / 表 | `図2` / `表1` |

- モデルへの指示仕様:
  - 本文に根拠がある主張には必ず根拠チップを付ける。可能な限り段落(¶)粒度まで特定する。
  - 本文から直接導けない内容は「**論文外の知識**」「**推測**」ラベルで区別する(§6)。
  - 本文に書かれていないことを問われたら「この論文には記載がない」と答えることを許容し、推測で埋めない。
- 根拠チップの検証: チップが指すブロック(節・段落・式・図表)が実在することをシステム側で検証し、壊れた参照は表示前に除去する。

## 6. メッセージ表現と AI 生成の明示

アシスタント回答は以下の要素で構成される(1a):

| 要素 | 仕様 |
|---|---|
| 発話者ラベル | 「✦ アシスタント」+ バッジ「**AI生成**」を回答ヘッダに常時表示 |
| 本文段落 | インライン根拠チップ入り。数式は KaTeX でレンダリング |
| 「論文外の知識」ボックス | 本文由来でない一般知識は、ラベルバッジ「論文外の知識」付きの淡色ボックスに分離して表示(例: 「実装では t を [0,1] から一様サンプリングし、ミニバッチでこの回帰を解くのが一般的です。」) |
| 「推測」ボックス | 本文から断定できない推論は同形式で「推測」ラベルを付す。決定: デザインには「論文外の知識」ボックスのみ描かれているが、免責文が「推測」表示を明記しているため、同一のボックス様式でラベルのみ変えて表示する(表現の一貫性のため) |
| 回答アクション行 | 「**↑ メモに保存**」「**再生成**」「**コピー**」の3操作 |

- 「再生成」は同じ質問での回答再生成。編集した質問の再送信にも対応する。
- 「コピー」は回答本文を Markdown としてクリップボードへコピーする(根拠チップはアンカー表記のテキストに展開)。

## 7. 定型アクション

チャット入力欄の上に定型プロンプトチップを**常設5種**並べる(1a 逐語)。すべて根拠チップ付きで回答される。

| チップ(常設) | 出力 |
|---|---|
| 3行要約 | 課題・手法・結果の3行 |
| 初心者向け解説 | 前提知識を補いながらの平易な説明 |
| 貢献と限界 | 論文の主張する貢献、明示された限界、暗黙の限界の指摘 |
| 実験設定の整理 | データセット・ベースライン・指標・ハイパーパラメータの表 |
| 実装の要点 | 再実装に必要な情報の抽出、擬似コード化(U2 向け) |

チップに常設しない定型アクションは、以下の導線に整理する(理由: 確定デザインのチップは5種であり、文脈が決まっているアクションは発生箇所に置くほうが1アクションで済む — P2):

| アクション | 導線 |
|---|---|
| 詳細要約 | 訳文モード(1b)の ✦3行要約カード内「詳細要約 →」から実行 |
| この式を説明 / この図を説明 | 数式・図ブロックのショートカット(§4、[04-viewer.md](04-viewer.md)) |
| 専門家向け要約 | 入力欄の入力候補(サジェスト)として維持。チップには出さない |
| 関連研究の位置づけ | 同上 |

- 決定: 「入力候補」は入力ボックスへのフォーカス時・入力途中に候補として提示する(デザインに UI は描かれていないため最小の実装とする。チップ行には追加しない)。
- 要約系の出力は論文カード・読了フロー(1g)の「✦ 要約をメモに保存」・記事生成([07-figures-and-articles.md](07-figures-and-articles.md))からも再利用される(同一内容を再生成しない)。

## 8. チャット → メモ昇格

- 各アシスタント回答の「↑ メモに保存」で、回答をメモとして保存する。保存されたメモは元メッセージと根拠アンカーへの参照を保ち、メモパネル・エクスポートに含まれる(S3)。
- スレッド全体の「まとめてメモ化」: スレッドを要点整理した 1 つの Markdown メモに変換する(変換結果は編集可能)。デザインには描かれていないが、長いスレッドの持ち出し(P5)に必要なため維持する。

## 9. 制約・ガードレール

- チャットは取り込んだ論文の内容に関する対話に限定しない(関連する一般知識の質問は自然な読解行動)が、コンテキストスコープは常に「この論文」であり、スコープチップで明示する(§2)。本文由来でない回答部分は必ず「論文外の知識」「推測」で区別する(§6)。
- レート・コスト: 1 メッセージごとの逐次ストリーミング表示。初回トークンまで 5 秒以内(p50)。長い論文でも文脈構築(§3)で入力を制御する。
- プロバイダ障害時はフォールバック連鎖(設定で変更可)。失敗した回答はエラーとして残し、勝手に消さない(P3)。
- チャット内容を LLM の学習に使わない旨をプライバシーポリシーに明記する([09-nonfunctional.md](09-nonfunctional.md))。

## 10. 受け入れ基準

- [ ] サイドパネルのチャットタブに「スレッド: メイン ▾」と「コンテキスト: この論文」チップが表示され、スレッドの切替・作成ができる
- [ ] 選択質問・「✦ この式を説明」で、選択箇所がチップ+引用プレビューとして表示され、回答後もチップから元位置へ飛べる
- [ ] チャットが参照中のブロックが本文側で強調され、「✦ チャットの根拠 · 式(5)」形式のバッジが付く
- [ ] 回答中の本文由来の主張に根拠チップ(段落粒度 `§2.1 ¶4` を含む)が付き、クリックで該当ブロックにジャンプ+一時ハイライトされる
- [ ] 実在しないブロックを指す根拠チップが表示前に除去される
- [ ] 回答に「AI生成」バッジが付き、本文由来でない内容が「論文外の知識」ラベル付きボックスで区別される
- [ ] 論文に記載のない事項を問うと、記載がない旨が明示される(でっち上げない)
- [ ] 入力エリアに免責文「回答は原文を根拠にします。本文にない内容は「論文外の知識」「推測」と表示されます。」が固定表示される
- [ ] 定型チップが5種(3行要約/初心者向け解説/貢献と限界/実験設定の整理/実装の要点)常設され、「実験設定の整理」で表形式の出力が得られる
- [ ] 「↑ メモに保存」でアシスタント回答をメモに昇格でき、昇格したメモは根拠アンカーへの参照を保つ
- [ ] チャット履歴が横断検索(4e)でヒットし、ヒット源「チャット」バッジ(紫)付きで「スレッドを開く」から該当スレッドへ遷移できる
