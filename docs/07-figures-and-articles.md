# 07. 概要図と記事モード

> 要旨: どちらも「理解のアウトプット」機能。**記事は独立したエディタではなく、ビューアの表示モードの1つ**であり(訳文 / 対訳 / 原文 / PDF / **記事** の5モード、[04-viewer.md](04-viewer.md))、訳文・メモ・チャット履歴を素材に AI が自動構成する読み物として提供する。図は二本立て: **全体概要図は構造化図データ→SVG の決定的レンダリング**、**詳細解説図・挿絵はラスター画像生成 API(プロバイダ切替可)**。AI 生成物には常に「AI生成」ラベルと根拠チップを付け、原文へ辿れるようにする(P1)。記事の公開・限定公開・コメントは v2 へ後ろ倒し。

## 1. 図の生成 — 二本立ての方式

### 1.1 方式の決定

| 対象 | 生成方式 | 理由 |
|---|---|---|
| **全体概要図**(記事冒頭・サムネイル) | 構造化図データ(中間 DSL)→ **SVG 決定的レンダリング** | 文字(日本語・数式)が正確に読めること、版管理・指示付き書き直し・SVG エクスポート・デザイントークン適用が本質要件。確定デザイン (1h) の 3 カードフロー図を 100% 同一に再現するには決定的描画が必須 |
| **詳細解説図・挿絵**(記事内の追加 2 枚程度) | **ラスター画像生成 API**(OpenAI gpt-image 系 / Google Gemini 画像 / xAI Grok imagine。プロバイダ切替可) | 概念のビジュアル化・挿絵は表現力優先。画像内テキストの破綻リスクは「重要テキストは画像内に置かずキャプション側に持つ」等の生成プロンプト仕様で緩和 |

- 更新経緯: 旧版の本ドキュメントは「ラスター画像生成モデルは使わない」としていたが、2026-07-06 のユーザー決定(画像生成に GPT / Gemini / Grok の API を使う)を受け、全体概要図=SVG を維持しつつ詳細解説図・挿絵にラスター生成 API を採用する二本立てに更新した。
- 画像生成プロバイダは `ImageProvider` 抽象化層で切り替える(テキスト生成の LLM プロバイダ切替と同様。[09-nonfunctional.md](09-nonfunctional.md))。既定・フォールバックは運営設定、ユーザーは設定 (4f) で変更できる。
- オプションとして、**概要図のラスター生成モード**(SVG の代わりに画像生成 API で描く)を設定で切替可能にする。既定は SVG。

### 1.2 全体概要図(1h デザイン準拠)

記事モードの冒頭に自動挿入されるブロック。**課題 → 提案 → 結果 の 3 カードフロー図**で、レイアウト・色はデザイントークンに従う(提案カード=アクセント強調、結果カード=グリーン)。

- カード構成: 各カードは「ラベル」(例:「課題」「提案 — RECTIFIED FLOW」「結果」)+「見出し」+「本文」を持つ。カード間は「→」で接続。
- ヘッダ行: 「✦ 全体概要図」+ バッジ「AI生成 · 版 2」+「✦ 書き直し指示」+「SVG ⤓」。
  - **版管理**: 再生成のたびに版が増え、前の版に戻せる。バッジに現在の版を表示する。
  - **✦ 書き直し指示**: 図に対する指示付き再生成(「実験の部分を削って手法を厚く」等)。
  - **SVG ⤓**: SVG 形式でダウンロード。エクスポートされた図にも AI 生成フッタが含まれる。
- フッタ行: 「✦ AI 生成 · Alinea · 2026-07-06」(生成主体と日付)+「根拠:」+ 根拠チップ(「§1」「§2.2」「表1」)。チップをクリックすると原文の該当セクション/図表へジャンプする。
- 図中の主張はすべて本文に根拠があること(生成プロンプト仕様)。数値(FID 等)は原文から転記し、照合チェックをかける。
- 決定: 旧版にあった「ラベルの直接編集」は持たない。理由: 確定デザインの操作は「書き直し指示 / 再生成 / SVG ダウンロード」のみで、記事モードを編集アプリ化しない方針(下記 §2.1)と整合させるため。手直しは指示付き書き直しで行う。
- 全体概要図はライブラリカードのサムネイルにも利用する(適切な図が論文にない場合の代替。[02-ingest.md](02-ingest.md))。

### 1.3 詳細解説図・挿絵

- 記事の生成・再生成時に、必要に応じて追加 2 枚程度の詳細解説図(手法パイプライン、直感的な比喩の挿絵など)をラスター画像生成 API で生成する。
- 生成プロンプト仕様:
  - 重要な情報(用語・数値・結論)は画像内テキストに依存させず、**キャプション側に持つ**。
  - サイトのトーン(落ち着いた低彩度、P6)に合わせたスタイル指示を共通プリアンブルとして与える。
- 生成画像にも「AI生成」バッジと根拠チップを付け、原論文の図の転載(§2.5)とは明確に区別する。
- 既定プロバイダはコストの安いモデル(例: Gemini の軽量画像モデル)とし、設定で変更・フォールバック連鎖できる。

### 1.4 生成タイミング

- 図の生成は記事モードの生成に付随する **オンデマンド** 処理。全論文で自動生成はしない(コストと、不要な論文が多いため)。
- 図単体の再生成はブロック単位の「✦ 書き直し指示」「再生成」で行う(記事全体の再生成を要しない)。

## 2. 記事モード(表示モードの1つ)

### 2.1 位置づけ

- 「記事」はビューアの **5 つの表示モードの 1 つ**(1h)。ヘッダのセグメント「訳文 | 対訳 | 原文 | PDF | 記事」で切り替える([04-viewer.md](04-viewer.md))。
- **独立エディタ・ブロックエディタは持たない。** 旧版の「ブロックエディタ文書」「1 論文に複数記事」は廃止し、**論文ごとに記事ビューは 1 つ**。手を入れたい場合はブロック単位の「✦ 書き直し指示」または記事全体の「✦ 指示つき再生成」で AI に指示する(編集アプリ化しない)。
- 記事モードでは左の目次サイドバーは幅 44px のアイコンレール(☰ / ブックマーク / 検索)に折り畳まれ、本文カラム(760px)に集中できる。

### 2.2 自動構成と免責表示

- 記事は **訳文・メモ・チャット履歴** を素材に AI がブログ風の読み物として自動構成する。ユーザーの読解の痕跡(疑問ハイライト、メモ、チャットでの議論)を反映した記事こそが定型要約との差別化点。
- 記事タイトルは AI 生成の日本語タイトル(例:「Rectified Flow を読む: 生成経路を「直線」にして 1 ステップ生成へ」)。
- メタ行に「AI生成」バッジと **生成日付**、免責文言を逐語で表示する:
  > 「訳文・メモ・チャット履歴から自動構成 · 2026-07-06 · 元の論文とは別物です — 根拠チップから原文へ」
- ヘッダに「✦ 指示つき再生成」ボタンを常設し、記事全体を指示付きで再生成できる(P1: 忠実性はチップで担保しつつ、構成はユーザーの指示で作り直せる)。

### 2.3 記事の構成ブロック

上から順に、記事は次のブロックで構成される(すべて AI が配置。ユーザーはブロック操作 §2.4 で調整):

| ブロック | 内容 | 明示要素 |
|---|---|---|
| 記事タイトル + メタ行 | AI 生成タイトル、生成日、免責文言 | 「AI生成」バッジ |
| 全体概要図 | §1.2 の 3 カードフロー図 | 「AI生成 · 版 N」、根拠チップ、SVG ⤓ |
| 見出し + 段落 | 訳文・メモ・チャットを再構成した本文(訳文本文と同じ Noto Serif JP) | ホバーツールバー(§2.4) |
| 原文引用ブロック | 英語原文をイタリックのセリフ体で引用。左にアクセント線 | 「原文引用」+ 位置チップ(例「§2.2 ¶3」)+「原文で見る →」 |
| 図表埋め込み | 原論文の図 + 日本語キャプション | 出典行 + ライセンスバッジ(§2.5) |
| 詳細解説図・挿絵 | §1.3 の生成画像 | 「AI生成」バッジ + キャプション |
| 議論したい点 | 番号付きリスト。AI が構成し、ユーザーの疑問ハイライト([04-viewer.md](04-viewer.md) の注釈色「疑問」)由来の項目には由来バッジを付ける | 「✦ AI構成」バッジ、「あなたの疑問ハイライトから」バッジ |
| 出典ブロック | 著者・論文名・発表先・arXiv ID・ライセンス | 「自動挿入 · 削除不可」バッジ |

- 出典ブロックは記事末尾に **自動挿入され、削除できない**(バッジ「自動挿入 · 削除不可」で操作対象外であることを明示)。
- 「議論したい点」は輪読会シナリオ(S4)の中核。疑問ハイライト由来と AI 発案の項目が混在し、由来バッジで区別される。

### 2.4 ブロック操作(ホバーツールバー)

各ブロックにホバーすると右上にダークの浮動ツールバーが現れ、3 操作を提供する:

1. **「✦ 書き直し指示」** — 指示を入力してそのブロックを書き直す(例:「もっと平易に」「式を使って」)。
2. **「再生成」** — 指示なしでそのブロックを再生成する。
3. **「根拠を表示」** — そのブロックの根拠となる原文箇所を提示する。

- AI 生成ブロックには **根拠チップ**(「§1」「§2.2」「表1」等、段落粒度まで)が付き、クリックで原文の該当位置へジャンプする(P1。チャットの根拠チップと同一の仕組み、[05-chat.md](05-chat.md))。
- 原文引用ブロックの「原文で見る →」は、引用箇所を原文表示モードで開く。

### 2.5 図表の転載とライセンス連動

記事への原論文図表の埋め込みは、**原論文ライセンスの可否判定を通る**([09-nonfunctional.md](09-nonfunctional.md) のマトリクス。判定結果はビューア情報パネルのライセンスカードにも表示: 「CC BY 4.0 — 図表転載可」「記事への図表埋め込み時、クレジットを自動付記します。」(2a))。

- **転載可(CC BY 等)**: 図を埋め込み、出典行に「出典: Liu et al., *Flow Straight and Fast* (arXiv:2209.03003)」+ ライセンスバッジ「CC BY 4.0 — 転載可」+「クレジット自動付記」を自動付記する(1h)。
- **転載不可(arXiv 既定の非独占ライセンス等)**: 埋め込み不可。代替として「原論文の図 N を参照(リンク)」カード、または概要図・解説図(自作図)への差し替えを提案する。

### 2.6 記事の構成プリセット(レベル別テンプレート)

記事の生成・再生成時に構成プリセットを選べる。プリセットは章立てと文体を規定する(記事ビューは 1 つのままで、プリセット変更=指示つき再生成の一種):

| プリセット | 章立ての骨子 | 特徴 |
|---|---|---|
| 初学者向け | 背景となる前提知識 → 何が課題か → 提案のアイデア(比喩・図中心)→ 何がうれしいか | 数式を避け、専門用語に注釈 |
| 実装者向け | TL;DR → 手法の構成要素 → 学習手順・損失・ハイパーパラメータ表 → 実装の落とし穴 → 再現チェックリスト | 擬似コード・表中心(U2) |
| 研究者向け | 位置づけ(何が新しいか)→ 手法の核心 → 実験の批判的読解 → 限界と展望 | 数式を含む詳細記述 |
| 輪読会向け | 発表フロー順(背景 → 課題 → 手法 → 実験 → 議論の論点) | 「議論したい点」を厚く構成(S4) |

- 「数式を避けた説明 / 数式を含む詳細説明」はプリセット属性(初学者=避ける、研究者=含む)+ 生成時トグルで制御する。
- 図表キャプションには補足説明(その図が記事文脈で何を示すか)を生成する。

### 2.7 導線

- **読了フロー (1g) から**: 読了ダイアログに「記事モードで読み返す →」カードがあり、読了直後に記事モードへ遷移できる([06-library.md](06-library.md))。記事未生成ならこの遷移が初回生成のトリガーになる。
- **横断検索 (4e) から**: 記事本文は横断検索のヒット源の 1 つ(グレーのバッジ)で、「記事モードで開く」で該当記事へ遷移する([06-library.md](06-library.md))。

### 2.8 共有・公開の扱い(v1 スコープ)

- **記事の公開・限定公開・コメント機能は v2 へ後ろ倒し**([10-roadmap.md](10-roadmap.md))。確定デザインに記事の公開 UI は存在せず、記事は個人の読み返しビューとして扱う。旧版 §2.4(可視性 3 段階・公開レイアウト・OGP・コメント)は本項に置き換え。
- **v1 の共有経路はコレクション共有ページ (4c) のみ**([06-library.md](06-library.md))。輪読会シナリオ(S4)は「コレクション共有リンク+各自が自分の記事モードで読み返す」で構成する。
- 決定: 記事本文はエクスポート対象に含める(全量 JSON 一括エクスポートの一部。P5「ロックインしない」)。理由: 設定 (4f) のエクスポート仕様は論文単位 Markdown / BibTeX・CSV / JSON 一括で、記事もユーザーの資産としてここに乗せる。記事単体の Markdown/HTML エクスポート UI は v1 では持たない(公開機能とともに v2 検討)。

## 3. 受け入れ基準

- [ ] 全体概要図の全テキストが正しくレンダリングされる(文字化け・意味不明語の混入ゼロ。構造化 SVG 方式で構造的に保証)
- [ ] 全体概要図が「課題 → 提案 → 結果」の 3 カードフローで描画され、「AI生成 · 版 N」バッジ・「✦ 書き直し指示」・「SVG ⤓」が機能する
- [ ] 概要図の再生成で版が増え、前の版に戻せる
- [ ] 詳細解説図が設定されたプロバイダ(gpt-image 系 / Gemini 画像 / Grok imagine)で生成でき、プロバイダを切り替えられる
- [ ] 詳細解説図の重要情報がキャプション側にあり、画像内テキストに依存していない
- [ ] 概要図のラスター生成モードを設定で切り替えられる(既定は SVG)
- [ ] 記事モードがビューアの表示モード切替(訳文/対訳/原文/PDF/記事)から開ける
- [ ] 構成プリセット(初学者向け/実装者向け/研究者向け/輪読会向け)を選んで記事を生成・再生成できる
- [ ] 記事メタ行に「AI生成」バッジ・生成日付・免責文言「元の論文とは別物です — 根拠チップから原文へ」が表示される
- [ ] 各ブロックのホバーで「✦ 書き直し指示」「再生成」「根拠を表示」が使える
- [ ] 根拠チップ・「原文で見る →」から原文の該当位置へジャンプできる
- [ ] 「議論したい点」で疑問ハイライト由来の項目に「あなたの疑問ハイライトから」バッジが付く
- [ ] 図表埋め込みがライセンス不可の論文でブロックされ、代替(リンクカード)が提示される。転載可の場合はクレジットとライセンスバッジが自動付記される
- [ ] 出典ブロックが常に記事末尾にあり、削除できない(「自動挿入 · 削除不可」)
- [ ] 読了フロー (1g) の「記事モードで読み返す →」から記事モードへ遷移できる
