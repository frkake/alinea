# 10. ロードマップと品質バー

> **要旨**: 「全部大事」は優先順位ではない。各マイルストーンを **ユーザー体験の言葉で書いた完成条件(DoD)** で区切り、前のマイルストーンの DoD を満たさずに次へ進まない。取り込みはブラウザ拡張(3a)に一本化されたため、**拡張機能は M0 に含める**(拡張がなければ論文が 1 本も入らないため)。品質レベルは A(LaTeX から完全構造化)/ B(PDF 由来)の 2 段階を全マイルストーンの前提とする。

## 1. マイルストーン方針

- DoD は実装項目の列挙ではなく「ユーザーに何が起きるか」で書く。DoD を満たさない限り次のマイルストーンに着手しない。
- 取り込み経路は拡張のみ([08-extension.md](08-extension.md))。したがって M0 の完成条件は「拡張から保存できること」から始まる。
- 品質レベルは A / B の 2 段階([02-ingest.md](02-ingest.md))。M0 は arXiv URL 経由の品質 A のみ、品質 B(PDF 直接送信)は M1 で追加する。
- 対象シナリオは [00-product.md](00-product.md) の S1〜S5 を参照する。

| マイルストーン | 一言でいうと | 対象シナリオ |
|---|---|---|
| M0 | 拡張で保存して、読んで、訊ける | S2 の主要部+S1 の取り込み部分 |
| M1 | 記録と日常運用が習慣として回る | S1・S3・S5 の完全達成 |
| M2 | 理解の深化とアウトプット(記事・図・共有・語彙) | S4 の完全達成 |
| M3 | 広がり(他ソース・学習ループ・オフライン) | 全シナリオの拡張 |

## 2. M0: コア読解体験(MVP)

**DoD: 「拡張から arXiv 論文を保存すると 1 分以内に読み始められ、数式・図表が崩れていない日本語で論文を読み通せ、選択して質問すると根拠つきで答えが返り、読みかけの位置が保存される。」**

- **拡張(3a・arXiv URL 経路)**: ツールバー→保存の 2 クリック。URL のみ送信し解析はサーバー。含める状態は 3 つ — 保存前(書誌プレビュー+「品質レベルA見込み」表示+ステータス 3 択[読む予定/すぐ読む/読んでいる]+タグ提案+ひとことメモ+Enter 保存)、保存直後(パイプライン: 書誌✓→構造化✓→翻訳中%)、既にライブラリ(重複検出。進捗・前回位置・「続きから開く」)。→ [08-extension.md](08-extension.md)
  - 決定: 保存前状態の「コレクション」選択はコレクション機能(M2)導入までフィールドを出さない。理由: 選択先が存在しないため。
  - 一般 PDF 状態(「このタブのPDFを送信」)とページ内「A 保存」ピルは M1。
- **取り込み・構造化**: arXiv HTML 優先(品質 A)、LaTeX フォールバックは M2 で主経路に逆転。メタデータ・ライセンス取得、arXiv ID による重複検知、品質レベルバッジ常時表示。構造化ドキュメント(中間表現)とブロック安定 ID。→ [02-ingest.md](02-ingest.md)、[01-domain-model.md](01-domain-model.md)
- **翻訳**: 自然訳スタイル、プレースホルダプロトコル+検証、段階的翻訳(書誌→アブスト→本文)、オンデマンド先行(未翻訳付録は「開くと翻訳します(オンデマンド)」)、共有キャッシュ。→ [03-translation.md](03-translation.md)
- **ビューア(1a/1b/1c)**: 表示モードは訳文/対訳/原文の 3 つ(PDF は M1、記事は M2)。対訳ポップ(キー t、「訳がおかしい?」→再翻訳導線)、KaTeX 数式、図表・引用のその場ポップオーバー(「図の位置へ移動→」「拡大」)、前回位置バナー(「続きから↓」)と位置保存、ダークモード。サイドパネルは M0 ではチャット/図表/情報の 3 タブ(メモ・注釈は M1、リソースは M2)。→ [04-viewer.md](04-viewer.md)
  - 決定: 参考文献一覧の表示は M0(図表タブ内)、「+この論文も取り込む」導線は M1。理由: M0 は拡張経路の安定化に集中し、アプリ内発の取り込みジョブを増やさない。
- **チャット**: メインスレッド、選択して「✦AIに質問」、¶ 粒度までの根拠チップ、「AI生成」バッジと「論文外の知識」ボックス、定型チップ(3行要約/初心者向け解説 など)。→ [05-chat.md](05-chat.md)
- **ライブラリ最小(1e/4a)**: 6 ステータス(読む予定/すぐ読む/読んでいる/読んだ/あとで再読/保留)、タグ、カード/テーブル 2 ビュー、クイックフィルタと基本ソート。保存フィルタ・一括操作は M2。→ [06-library.md](06-library.md)
- **アカウント**: メール+OAuth。BYOK 設定の受け皿(LLM プロバイダ切替の既定は運営キー)。→ [09-nonfunctional.md](09-nonfunctional.md)

M0 に含めない主なもの: 注釈・メモ(M1)、通知(M1)、PDF(M1)、記事・概要図・語彙帳・リソース・コレクション(M2)。

## 3. M1: 記録と日常運用

**DoD: 「毎朝のトリアージと読了後の記録が習慣として回り、数ヶ月前に読んだ内容を横断検索で思い出せる。arXiv にない PDF も拡張から送って読める。」**

- **注釈**: 4 色ハイライト(重要/疑問/アイデア/用語)+コメント+ブックマーク、注釈一覧パネル(フィルタ「すべて/重要/疑問/アイデア/コメントのみ」+未配置の保全)。→ [04-viewer.md](04-viewer.md)
- **メモ**: メモタブ、チャット回答の「↑メモに保存」昇格。
- **読了フロー(1g)**: 理解度 1〜5・重要度低/中/高・ひとことメモ(「何に使えるか」)、「✦要約をメモに保存」、すべてスキップ可。→ [06-library.md](06-library.md)
  - 決定: モーダル内の「記事モードで読み返す→」カードは M2(記事モード導入)まで非表示。理由: 遷移先が存在しないため。
- **ダッシュボード(1d)**: 続きを読む/すぐ読むキュー(ドラッグ順序、6 本で「積みすぎかも?」)/最近追加(パイプライン進捗・✦3行要約)/控えめな統計。締切カードはコレクションと同時に M2。→ [06-library.md](06-library.md)
- **通知(4a)**: 通知アイコン(◷)+ポップオーバー。翻訳完了(「読み始める→」)とステータス変更提案(「変更する・そのまま」— 自動適用しない、P6)。締切リマインドはコレクション締切と同時に M2。→ [06-library.md](06-library.md)
- **横断検索(1e/4e)**: 検索ドロップダウン+専用結果画面。ヒット源明示(本文/訳文/メモ/チャット。記事は M2 で追加)・日英クロス。→ [06-library.md](06-library.md)
- **用語管理と部分再翻訳**: 3 層用語集(訳語統一用。語彙帳とは別物)、訳語変更→影響ブロックの部分再翻訳、手動修正・指示つき再翻訳。→ [03-translation.md](03-translation.md)
- **読書時間記録**: 計測 ON/OFF は設定(4f)。統計は控えめ(P6)。
- **エクスポート**: 論文単位 Markdown(Obsidian 互換)・BibTeX。CSV/JSON 一括は M2(設定 4f のエクスポート節を完成させるタイミング)。
- **拡張の残り 2 要素**: 一般 PDF の直接送信(品質 B、「書誌は推定」の明示、private 既定)+ページ内「A 保存」ピル(オプトイン)。→ [08-extension.md](08-extension.md)
- **PDF 表示モード(2a)**: 原紙面⇄構造化ビュー相互リンク(「同期: p.5 ≒ §2.2」)、情報パネル(取り込みタイムライン・再取り込み・ライセンスカード)。
  - 決定: PDF モードは PDF 直接送信(品質 B)と同じ M1 に置く。理由: 品質 B では原紙面との突き合わせが「黙って壊れない」(P3)ための逃げ道になるため、B 導入と同時が必須。

## 4. M2: 理解の深化とアウトプット

**DoD: 「読了した論文を概要図つきの記事モードで読み返し、コレクションを URL 共有して輪読会に臨める。LaTeX ソース由来の最高品質(品質 A 主経路)で読め、拾った単語が復習で定着する。」**(概要図は記事モード側=個人閲覧の要素であり、共有ページには載らない — [09-nonfunctional.md](09-nonfunctional.md) §5)

- **記事モード(1h)**: 5 つ目の表示モードとして追加(独立エディタではない)。訳文・メモ・チャット履歴からのブログ風再構成、レベル/テンプレート選択、ブロックホバーで「✦書き直し指示/再生成/根拠を表示」、根拠チップ、原文引用ブロック、「議論したい点」、出典ブロック(削除不可)、ヘッダ「✦指示つき再生成」。公開機能は含まない(v2 検討 — §5 末尾参照)。→ [07-figures-and-articles.md](07-figures-and-articles.md)
- **全体概要図**: 課題→提案→結果の 3 カードフロー。構造化図データ→SVG 決定的レンダリング(100% 同一デザイン)。版管理(「AI生成 · 版 2」)・指示つき書き直し・SVG ダウンロード。→ [07-figures-and-articles.md](07-figures-and-articles.md)
- **解説図(画像生成 API)**: 記事内の詳細解説図・挿絵をラスター画像生成(GPT / Gemini / Grok、プロバイダ切替可)。概要図のラスター生成モードは設定で切替可能なオプション。重要テキストはキャプション側に持つ生成プロンプト仕様で文字破綻を緩和。→ [07-figures-and-articles.md](07-figures-and-articles.md)
- **コレクション(4b)+共有ページ(4c)**: 順序ドラッグ・締切+残日数・担当者割当・読了進捗。共有リンク(発行/無効化、「共有ページにメモを含める」トグル)。共有ページはアカウント不要・閲覧専用・noindex。締切リマインド通知と、ダッシュボードの締切カードもここで有効化。→ [06-library.md](06-library.md)
- **LaTeX パーサ主経路化**: arXiv LaTeX ソース解析を品質 A の主経路に昇格。既存 B 論文の再取り込みによる A 昇格とリアンカー。→ [02-ingest.md](02-ingest.md)
- **直訳スタイル**: 自然訳との切替(段落・セクション単位)。→ [03-translation.md](03-translation.md)
- **語彙帳(4d)完全版**: 選択メニュー「語彙に追加」での文脈ごと保存+一覧(単語/コロケーション/イディオム)+詳細(語義・語源・解釈のしかた・覚えるコツを AI 自動生成、編集可)+SRS 復習(忘却曲線、「まだあやしい」/「✓覚えた」、「原文で見る→」)。旧計画の M3 から前倒し(確定デザインで第一級画面になったため)。→ [11-vocabulary.md](11-vocabulary.md)
- **リソースタブ(5a)**: GitHub 実装/発表動画/スライド/解説記事の添付。URL 貼り付け→種別自動判定、arXiv 由来の公式実装自動検出(「公式実装」バッジ)。→ [12-resources.md](12-resources.md)
- **ライブラリ強化**: 属性フィルタ+「この条件を保存」(保存フィルタ)、複数選択→フローティング一括操作バー(ステータス変更/タグ追加/コレクションへ)、優先度・締切列。→ [06-library.md](06-library.md)
- 横断検索のヒット源に「記事」を追加。エクスポートに CSV/JSON 一括を追加。

## 5. M3: 広がり

**DoD: 「arXiv 以外の主要ソースの論文も同じ体験で読め、学習・再読のループがツールの外(Anki・オフライン)まで届く。」**

- **他サイトアダプタ**: OpenReview / ACL Anthology / PubMed / PMC。Hugging Face Paper PageはarXiv同定とGitHub／project／Model／Dataset／Space候補の収集に使う。→ [02-ingest.md](02-ingest.md)、[12-resources.md](12-resources.md)、[08-extension.md](08-extension.md)
- **GitHubコード対応解析**: 論文blockと公開リポジトリのfile／symbol／行範囲を対応付ける。`off` / `on_demand` / `automatic`、月額予算、実行前見積もりでAPI費用を制御する。→ [12-resources.md](12-resources.md)
- **OCR**: スキャン PDF への対応検討。品質レベルは新設せず B の内部拡張として扱う(A/B の 2 段階は維持)。
- **arXiv バージョン差分表示**: v1→v2 の変更点提示とリアンカー。
- **Anki エクスポート**: 語彙帳からのカード書き出し。→ [11-vocabulary.md](11-vocabulary.md)
- **やさしい訳スタイル**: 第 3 の翻訳スタイル。→ [03-translation.md](03-translation.md)
- **セマンティック検索**: 類似手持ち論文・クエリ翻訳のクロス検索強化。
- **輪読会スペース**: グループ共有(Q4 の判断後)。v1 はコレクション共有リンク+各自の記事モード読み返しで代替する。
- **PWA オフライン閲覧**。
- **(M3 スコープ外・v2 検討)記事の公開/限定公開/コメント**: 確定デザインに公開 UI が存在しないため v1(M0〜M3)スコープ外。v1 の共有経路はコレクション共有ページ(4c)のみ。→ [07-figures-and-articles.md](07-figures-and-articles.md)

## 6. 継続的な品質指標(全マイルストーン共通で計測)

| 指標 | 目標 | 意味 |
|---|---|---|
| arXiv 取り込み成功率(HTML/LaTeX 存在時の品質 A 率) | ≥ 99% | 入口の信頼 |
| 拡張保存→readable 到達時間 p50 | ≤ 60 秒 | M0 DoD の「1 分以内」 |
| プレースホルダ検証通過率 | ≥ 99.9% | 数式・参照の無破壊 |
| 数式レンダリング成功率(品質 A) | ≥ 99% | 読解体験の底 |
| 「訳がおかしい?」報告率 | 低下傾向を維持 | 翻訳品質の代理指標 |
| LLM コスト/論文(用途別ルーティング後) | 予算内([09-nonfunctional.md](09-nonfunctional.md)) | 運営持続性。安価モデル(翻訳)と上位モデル(チャット・記事)の配分を実測 |
| 週次リテンション(2 本以上読むユーザー) | 追跡 | 習慣化の代理指標 |

## 7. 主要リスクと対応

| リスク | 影響 | 対応 |
|---|---|---|
| 取り込みの拡張一本化(ストア審査遅延・Chrome/Edge 以外のユーザー) | 保存できない=製品が始まらない | 拡張を M0 のクリティカルパスに置き、審査期間をスケジュールに織り込む。Chrome/Edge 両対応を M0 から |
| arXiv HTML のカバレッジ限界(古い論文・変換失敗) | 品質 A 率低下 | LaTeX パーサ(M2)で主経路を置換。フォールバック連鎖は M0 から |
| PDF 解析品質 | 品質 B 体験の不満 | ベストエフォートを明示(品質バッジ常時表示+「書誌は推定」)。PDF モードで原紙面に逃がし(P3)、再取り込みで A へ昇格 |
| LLM コスト超過 | 運営持続性 | 共有キャッシュ・オンデマンド生成・クォータ([09-nonfunctional.md](09-nonfunctional.md))に加え、用途別プロバイダルーティング(翻訳=安価モデル/チャット・記事=上位モデル)と BYOK。早期に実測 |
| 特定 LLM プロバイダの障害・モデル廃止 | 翻訳・チャット停止 | GPT/Claude/Gemini/DeepSeek の切替可能設計+フォールバック連鎖。モデル指定は設定で変更可能に保つ |
| 画像生成のテキスト破綻(解説図) | 図の信頼低下 | 全体概要図は SVG 決定的レンダリングを維持し画像生成に依存しない。解説図は「重要テキストはキャプション側に持つ」プロンプト仕様で緩和 |
| 著作権クレーム | サービス停止リスク | ライセンスマトリクス([09-nonfunctional.md](09-nonfunctional.md))を M0 から実装。共有系(4c)は M2 まで出さず、記事公開は v2 検討に後ろ倒し |
| 翻訳の幻覚・省略 | 信頼喪失(致命的) | 自動品質検査+対訳ポップ+原文フォールバック(P1/P3)。「訳がおかしい?」導線を M0 から |

## 8. 受け入れ基準チェックリスト

- [ ] M0: 拡張から arXiv 論文を保存すると 1 分以内に読み始められる(保存前に「品質レベルA見込み」とステータス 3 択、保存直後にパイプライン進捗、保存済みなら「続きから開く」)
- [ ] M0: 訳文/対訳/原文の 3 モードで数式・図表が崩れず読み通せ、品質バッジが常時見える
- [ ] M0: 選択して「✦AIに質問」すると根拠チップつきで答えが返り、「AI生成」「論文外の知識」が区別されている
- [ ] M0: ブラウザを閉じても読みかけの位置に「続きから↓」で戻れる
- [ ] M1: 朝のトリアージ(ダッシュボード+通知)と読了フロー(1g)が回り、ステータスは提案のみで自動変更されない(P6)
- [ ] M1: 数ヶ月前に読んだ内容を横断検索(日英クロス・ヒット源明示)で 2 分以内に見つけられる(S5)
- [ ] M1: arXiv にない PDF を拡張から送信して品質 B バッジつきで読め、PDF モードで原紙面と突き合わせられる
- [ ] M2: 読了論文を記事モードで読み返せ、全体概要図(SVG・版管理)がダウンロードできる
- [ ] M2: コレクション共有リンクをアカウント不要・noindex の共有ページ(4c)として開ける
- [ ] M2: 「語彙に追加」した語が SRS 復習に現れ、「原文で見る→」で文脈に戻れる
- [ ] M2: LaTeX ソース由来(品質 A 主経路)で取り込まれ、既存 B 論文が A へ昇格できる
- [ ] M3: arXiv 以外の対応ソースでも M0 と同じ読解体験が成立する
- [ ] 全期間: §6 の品質指標を計測し、目標を下回るマイルストーンはリリースしない
