# 04. 論文ビューア

> 要旨: ユーザーが最も長い時間を過ごす画面。「読む流れを切らない」ことを最上位の体験要求とする(P1, P6)。表示モードは **訳文 / 対訳 / 原文 / PDF / 記事** の5つ、サイドパネルは **チャット / メモ / 注釈 / 図表 / リソース / 情報** の排他6タブ。本書は確定デザイン (1a)(1b)(1c)(2a)(1h)(5a) を正とする。

## 1. 画面構成

デスクトップ(基準ビューポート 1440×900)は 3 ペイン構成:

```
┌──────────┬──────────────────────────────┬────────────────┐
│ 目次      │ 本文                           │ サイドパネル      │
│ 232px    │ 表示モード:                     │ 320〜340px      │
│ (44px    │ 訳文/対訳/原文/PDF/記事          │ チャット/メモ/注釈 │
│ レールに  │                                │ /図表/リソース/情報│
│ 折畳可)   │                                │ (排他タブ)       │
└──────────┴──────────────────────────────┴────────────────┘
```

- **ヘッダ**(h=52px): 左から「‹」(戻る)/ 論文タイトル(1行省略)/ 品質バッジ「A」(ツールチップ「品質レベルA: LaTeXソースから完全構造化」)/ ステータスピル(色ドット+「読んでいる ▾」等、ドロップダウンで変更)/ 表示モード切替セグメント / 「スタイル: 自然訳 ▾」(翻訳スタイル切替、[03](03-translation.md))/ 論文内検索「この論文内を検索」(キーキャップ「/」表示)/ オーバーフローメニュー「⋯」。記事モードではスタイルセレクタの位置に「✦ 指示つき再生成」が入る (1h)。
- **目次ペイン**(w=232px): セクションツリー(章/節2階層)。現在位置ハイライト(アクセント淡背景+左2pxバー)、節ごとの翻訳済み「✓」、注釈数バッジ、ブックマーク印、ヘッダに翻訳進捗「翻訳 96%」と折りたたみ「⟨⟨」。未翻訳の付録は破線ボックスで「付録 A 証明 — 未翻訳」「開くと翻訳します(オンデマンド)」と表示し、開くとオンデマンド翻訳が走る。フッタに「今日の読書 42分」「位置は自動保存」。
- **目次レール**(w=44px): 訳文・記事モードの既定では目次を 44px のアイコンレール(☰ / ブックマーク / 検索)に折り畳む (1b)(1h)。☰ で展開。
- **本文ペイン**: 下記 §2 の表示モード。
- **サイドパネル**: §10。幅はデザイン準拠で 320px(訳文ゆったり 1b)〜340px(対訳・PDF・リソース 1a/2a/5a)。

## 2. 表示モード(5種)

ヘッダのセグメンテッドコントロールでワンクリック切替。

| モード | 内容 | 主用途 |
|---|---|---|
| 訳文 | 日本語1カラム(ゆったり)。段落ホバー「対」+キー `t` で対訳ポップ | 通常読解 (1b) |
| 対訳 | 左=原文 / 右=訳文の段落単位2カラム。「段落対応 ⇄」 | 精読・英語学習 (1a)(1c) |
| 原文 | 構造化 HTML の英語表示 | 原文精読 |
| PDF | オリジナル PDF レンダリング+構造化ビュー相互リンク | レイアウト確認・品質 B の論文 (2a) |
| 記事 | 訳文・メモ・チャット履歴から AI がブログ風に再構成した読み物+全体概要図 | 読み返し・整理 (1h)。仕様は [07-figures-and-articles.md](07-figures-and-articles.md) |

- 未翻訳ブロックは原文+控えめな「翻訳中…」表示。翻訳失敗ブロックは原文+理由表示(P3)。
- 品質レベルは A / B の2段階([02](02-ingest.md))。PDF モードの相互リンクは両レベルで提供し、品質 A は構造化ビュー側が主、品質 B は PDF 側が主になる。

## 3. 訳文モード(1b)

- **ゆったりレイアウト**: 本文カラム幅 720px 中央寄せ(サイドパネル 320px の 1b。サイドパネル 340px の画面 5a では 680px)、本文 16.5px / 行間 2.15、書体 Noto Serif JP(設定でゴシックに切替可、`--pr-jp`)。
- **前回位置バナー**: 本文上部中央に浮遊ピルで「前回はここまで: §3.1 実験 · 昨日 21:52」+ボタン「続きから ↓」+「×」。即座に自動ジャンプはしない(冒頭を確認したい時もあるため。§11)。
- **✦ 3行要約カード**: アブストラクト冒頭に AI 生成の3行要約を番号付きで表示。「AI生成」バッジと「詳細要約 →」導線(詳細要約はチャットへの定型導線、[05](05-chat.md))。
- **見出しの原題併記**: 「1 はじめに — Introduction」のように日本語見出しに原語をイタリック(Source Serif 4)で併記。
- **対訳ポップ**: 段落ホバーで左マージンに「対」ボタンが出現、クリックまたはキー `t` で段落直下に原文をインライン展開する。「この訳、合ってる?」を 1 秒で確認できることが信頼の要(P1)。
  - ポップのヘッダに段落位置「¶2 / 1 Introduction」と「閉じる ×」。
  - フッタに「訳がおかしい?」→「再翻訳」「指示つき再翻訳」(段落単位の訳し直し、[03](03-translation.md))と「t で開閉」のキーヒント。
  - ポップ内の原文英語に対しても選択メニュー(§8)が使える。

## 4. 対訳モード(1a)(1c)

- 段落単位の2カラムグリッド(左=原文 Source Serif 4 13.8px/1.72、右=訳文 Noto Serif JP 14.8px/2.0)。太字等のインライン強調は原文・訳文で対応して保持される。
- カラム見出し: 「原文 — ENGLISH」/「訳文 — 自然訳 ✦ AI翻訳」+右端に「段落対応 ⇄」(段落対応の可視化・スクロール同期)。
- 訳文は原文と文数が一致しないことがある(自然訳)が、対応は段落粒度で保証する。
- **数式ブロック**: 式番号付き・全幅表示。直下にアクション「✦ この式を説明」(式をコンテキストにチャットへ送る、[05](05-chat.md))と「LaTeX をコピー」。
- **チャット根拠の本文側強調**: チャットが根拠として参照中のブロックは本文側でアクセント淡背景+枠で強調され、「✦ チャットの根拠 · 式(5)」バッジが浮く(チャット⇔本文の双方向リンク)。
- 決定: 画面幅 1200px 未満では「段落交互(原文の直下に訳文)」レイアウトに自動で切り替える。理由: 2カラムの最小可読幅を割るため(デザインはデスクトップのみ提示。旧仕様を維持)。

## 5. PDF モード(2a)

原紙面のページレンダリングと構造化ビューを `page+bbox` で相互リンクする。

- **ツールバー**: ページナビ「‹ 5 / 24 ›」(番号直接入力可)/ ズーム「− 128% +」/ フィット「幅に合わせる ▾」/「見開き」トグル / 同期インジケータ「同期: p.5 ≒ §2.2 Reflow」(ページ⇔節の近似対応を常時表示)/「この位置を訳文で開く →」。
- **bbox 選択**: PDF 本文の段落を bbox 単位で選択すると「≒ §2.2 ¶2 — 訳文で見る →」チップが出現し、クリックで訳文の対応段落へジャンプする。
- **左サイドバー**: 「目次 / ページ」セグメントで切替。ページサムネイル(現在ページはアクセント枠+ハイライト位置の示唆)、フッタに「24 ページ · 4.1 MB」「⤓ 原文PDF」。
- サイドパネル(情報タブ)の内容は §10.6。

## 6. 記事モード(1h)

論文をブログ風の読み物として AI が再構成する表示モード(独立エディタではない)。冒頭の全体概要図・ブロックホバーの「✦ 書き直し指示 / 再生成 / 根拠を表示」・原文引用ブロック・出典/ライセンス自動付記を含む仕様は [07-figures-and-articles.md](07-figures-and-articles.md) に委譲する。

## 7. 本文内のインタラクション

読む流れを切らないため、参照類は **その場に展開** する(スクロールしない)。

- **図表参照**(「図2」「表1」): クリックでその場にポップオーバー表示 (1c)。内容は図(原論文の画像)+原文キャプション+訳キャプションの両言語併記+アクション「図の位置へ移動 →」「拡大」「✦ この図を説明」(AI、[05](05-chat.md))。参照中のテキストはアクセント強調され、図表タブの該当カードと「(表示中)」で同期する。
- **数式参照**(「式(5)」): 同様にポップオーバー。数式ブロック自体のアクションは §4。
- **引用**(「[12]」等): 原文・訳文の両方でアクセント色リンク。決定: クリックで図表タブの「参考文献」該当項目を展開表示する(§10.5)。理由: デザインは本文のリンク様式と参考文献側の展開状態(取り込みアクション付き)を提示しており、両者を接続するのが最短動線のため。
- **脚注**: インライン展開(品質 A は脚注を構造として保持、[02](02-ingest.md))。
- **数式表示**: KaTeX レンダリング。番号付き数式はアンカーを持ち、「LaTeX をコピー」でソースを取得できる。
- **図**: ポップオーバーの「拡大」でライトボックス(原寸・キャプション対訳)。遅延ロード。

## 8. テキスト選択メニュー(1b)

本文(原文・訳文どちらでも、対訳ポップ内の原文でも)を選択すると、ダークのフローティングツールバーを表示する:

`● ● ● ●(4色ハイライト) | コメント | ✦ AIに質問 | 語彙に追加 | コピー`

- **4色ドット**: 選択範囲をその色でハイライト(§9 の注釈4色)。
- **コメント**: ハイライトにテキストを付ける。
- **✦ AIに質問**: 選択範囲をアンカー(引用チップ+引用テキスト)付きでチャットに送る([05](05-chat.md))。
- **語彙に追加**: 単語/コロケーション/イディオムを文脈センテンスごと語彙帳へ保存(語義・語源・解釈のしかた・覚えるコツは保存時に AI 生成)。**原文(英語)選択時のみ有効**で、訳文選択時は非活性(対訳ポップの原文選択では有効)— [11-vocabulary.md](11-vocabulary.md) の登録入口の規定と一致。
- **コピー**: 引用形式(テキスト+論文タイトル・セクション参照)/プレーンを選べる。

## 9. 注釈(Annotation)

- **ハイライト4色**(意味と正確な色):

| 色 | 意味 | hex |
|---|---|---|
| アンバー | 重要 | `#C49432` |
| ブルー | 疑問 | `#5884AA` |
| グリーン | アイデア | `#659471` |
| グレー | 用語(無ラベル) | `#82827E` |

  本文上の描画は低彩度の半透明背景(例: 重要=rgba(196,148,50,0.26))で読書の邪魔をしない(P6)。
- **注釈番号チップ**: ハイライトの直後に丸数字チップ(例「①」相当の 14px 丸バッジ)が付き、注釈一覧と連動する。
- **コメント**: ハイライトに任意のテキスト(💬)を付けたもの。
- **ブックマーク**: セクション単位のしおり。目次にしおりアイコンで表示。
- 注釈はすべて原文ブロックにアンカーされ([01](01-domain-model.md))、翻訳の再生成・スタイル切替・リビジョン昇格を跨いで保持される。位置を失った注釈は「未配置」として注釈一覧に残す(黙って消さない、P3)。
- **注釈一覧パネル**(サイドパネル「注釈」タブ、1b): フィルタチップ「すべて 6 / 重要 3 / 疑問 2 / アイデア 1 / コメントのみ」(件数付き)。各カードは色バー+引用テキスト+コメント+メタ「§2.1 整流フロー · 昨日 21:40」。クリックで本文該当位置へジャンプ。フッタに「未配置 0 件」と「⤓ Markdown エクスポート」(P5)。

## 10. サイドパネル(排他6タブ)

タブは排他選択・件数バッジ付き(例「注釈 6」「リソース 4」)。決定: 全表示モードで共通に開ける(記事モード 1h はパネルを閉じた状態のみ提示。開閉可能とする)。

| タブ | 内容 | 詳細仕様 |
|---|---|---|
| チャット | 読解チャット(スレッド・根拠チップ・定型チップ) | [05-chat.md](05-chat.md) |
| メモ | 論文単位の自由メモ。チャット回答の「↑ メモに保存」先 | [05-chat.md](05-chat.md) |
| 注釈 | 注釈一覧(フィルタ/未配置/エクスポート) | §9 |
| 図表 | 図表一覧+参考文献 | §10.2 |
| リソース | GitHub実装・動画・スライド・解説記事の添付 | [12-resources.md](12-resources.md) |
| 情報 | 書誌・品質と取り込み・ライセンス・エクスポート | §10.3 |

### 10.1 チャットタブ(1a)

ビューア文脈での要点のみ(全仕様は [05-chat.md](05-chat.md)): スレッド切替「スレッド: メイン ▾」、コンテキストチップ「コンテキスト: この論文」。AI 回答は「AI生成」バッジ+インライン根拠チップ(§ / 式 / 図 / 段落 `¶` 粒度まで)を持ち、チップクリックで本文該当箇所へジャンプ+強調(§4)。本文に根拠がない内容は「論文外の知識」ボックスで区別。定型チップ5種「3行要約 / 初心者向け解説 / 貢献と限界 / 実験設定の整理 / 実装の要点」。免責文「回答は原文を根拠にします。本文にない内容は「論文外の知識」「推測」と表示されます。」を入力欄下に固定表示。

### 10.2 図表タブ(1c)

1パネル内に2セクション:

- **図表一覧**: 各項目=サムネイル+キャプション+位置「§2.2 · p.5」。本文で参照表示中の図はハイライトされ「(表示中)」と付記(本文ポップオーバーと同期)。クリックで該当図を表示。
- **参考文献**: 各項目=引用番号(等幅)+著者+タイトル+venue・年。行クリックで展開し、「+ この論文も取り込む」(参照文献をワンクリックでライブラリへ取り込む。読解中に芋づるで積める)/ 既存の場合は「ライブラリに有り ✓」/ arXiv 外部リンクを表示。

### 10.3 情報タブ(2a)

- **書誌情報**: タイトル / 著者 / venue(ICLR 2023)/「arXiv:2209.03003 ↗」「DOI ↗」外部リンクチップ。
- **品質レベルと取り込み**: バッジ+説明文(A=「LaTeX ソースから完全構造化。数式・相互参照・図表・脚注を保持しています。」)。取り込みタイムライン3段(例: 「arXiv から LaTeX ソース取得」→「構造化・図表抽出(24p / 図8 / 表4)」→「全文翻訳 完了(自然訳 · v3)· 付録は未翻訳」、各タイムスタンプ付き)。アクション「再取り込み」「処理ログ」。
- **ライセンス**: 判定結果カード(例「CC BY 4.0 — 図表転載可」「記事への図表埋め込み時、クレジットを自動付記します。」)。記事モードの転載可否と連動([07](07-figures-and-articles.md))。
- **エクスポート**: 「注釈 Markdown ⤓」「原文 PDF ⤓」(P5)。
- フッタ注記: 「読書時間を記録しています(設定でオフにできます)」。

## 11. 読解位置と読書時間

- スクロール位置(先頭可視ブロック)を自動保存し(「位置は自動保存」)、次回開いたとき前回位置バナー(§3)で復帰できる。デバイス間で同期する。
- ReadingSession としてアクティブ時間(タブ前面かつ操作/スクロールあり)を記録し、目次フッタに「今日の読書 42分」を表示。統計([06](06-library.md))と「読んでいる」への遷移提案に使う。計測は設定に明記し、オフにできる。

## 12. 論文内検索

ヘッダの検索(ショートカット `/`)は原文・訳文の両面を対象とする。訳文ヒットは対応原文ブロックと同一視する(横断検索の同ルールと整合、[06](06-library.md))。決定: ヒットは目次にマーカー表示する(デザイン未記載の補完。変更があればデザイン優先)。

## 13. モバイル / タブレット

- v1 のモバイルは**閲覧とステータス変更のみ**([00-product.md](00-product.md) スコープ)。「訳文モード+タップ対訳」を既定とし、サイドパネルはボトムシート化、目次はドロワー。取り込みは拡張機能のみのため([02](02-ingest.md))モバイルからの取り込みはない。
- 注釈作成・チャット送信・語彙追加などの操作系のモバイル完全対応は M3 で拡張する(確定デザインはデスクトップ 1440×900 のみのため、v1 では読み特化に縮退)。
- PWA としてインストール可能にする。オフラインは「開いたことのある論文の閲覧」のみベストエフォート(M3)。

## 14. 表示設定・アクセシビリティ

- フォントサイズ・行間・本文幅の調整。本文書体は明朝(Noto Serif JP)⇄ゴシック(IBM Plex Sans JP)切替(`--pr-jp`)。アクセントカラー4色切替。設定画面(4f)の全体構成とデザイントークンは [09-nonfunctional.md](09-nonfunctional.md) を参照。
- **ダークモード**あり (1c): 基調 `#181B20` 系(パネル #1E2228 / 面 #22262D / 境界 #2A2F37 等)。アクセントはライト用 `--pr-a` の単純流用ではなく、ダーク用対応色 `--pr-ad`(既定 #8FAECB)/ `--pr-ads` / `--pr-adm` を使う。ライト/ダーク/システム追従。
- キーボードショートカット: `/` 論文内検索、`t` 対訳ポップ開閉(いずれもデザインで確定。UI 上にキーヒント表示)。決定: `j/k` 段落移動、`m` 表示モード循環、`c` チャット、`b` ブックマークを併せて提供する。理由: デザインには未記載だが読書系ツールの慣習であり、マウス往復を減らす(変更があればデザイン優先)。
- 数式には LaTeX ソースを代替テキストとして付与する。

## 15. 受け入れ基準

- [ ] ヘッダで 訳文/対訳/原文/PDF/記事 の5モードをワンクリックで切替できる
- [ ] サイドパネルが排他6タブ(チャット/メモ/注釈/図表/リソース/情報)で、注釈・リソースに件数バッジが出る
- [ ] 訳文モードで任意の段落の原文を 1 操作(ホバー「対」またはキー `t`)で確認でき、ポップに段落位置(¶)と「訳がおかしい?」導線がある
- [ ] 再訪時に前回位置バナー「続きから ↓」が表示され、1クリックで前回位置(§)へ復帰できる
- [ ] 「図2」「式(5)」をクリックしてもスクロール位置を失わない(その場ポップオーバー+両言語キャプション)
