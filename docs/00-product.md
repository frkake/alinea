# 00. プロダクト定義 — Alinea

> 本書は仕様全体の背骨である。個別機能の仕様で判断に迷ったら、この文書の「プロダクト原則」と「主要シナリオ」に立ち返って決める。本書を含む全 docs は確定デザイン(`論文読解システム デザイン.dc.html`)を正とし、旧記述と矛盾する場合はデザインに合わせて更新済みである。

## 1. プロダクトの位置づけ

**Alinea — 論文読解ワークベンチ**。英語論文を日本語で **読む行為そのもの** を支援する研究者向けツールである。翻訳ツールでも文献管理ツールでもなく、その中間にある「読解」に焦点を置く。対訳を軸に、翻訳・注釈・読解チャット・語彙学習・外部リソース参照までを 1 つのワークベンチにまとめる。

- 翻訳サービス(DeepL 等)との違い: 図表・数式・引用構造を保った論文専用の構造化翻訳であり、訳文の上で質問・記録・共有ができる。
- 文献管理ツール(Zotero 等)との違い: 書誌管理や執筆連携は目的ではない。「読む前(トリアージ)→ 読んでいる最中(理解)→ 読んだ後(記録・想起・共有)」のワークフローを持つ。
- alphaXiv 等との違い: 論文ページへの公開ディスカッションではなく、個人の読解と理解の蓄積が主。「記事」は第一にビューアの表示モード(自分の読解を読み物として読み返す)であり、任意で公開スナップショットとして共有・コメント受付もできる(P7 整合として公開版には原文引用本文・訳文・メモ・チャット・原論文図を含めない。→ §5・[07-figures-and-articles.md](07-figures-and-articles.md))。

通し例の題材論文は実在の Rectified Flow(arXiv:2209.03003)。デザイン・シードデータ・本 docs の例示はすべてこれで統一する。

## 2. プロダクト原則

仕様上のすべての判断は、以下の原則に従う。番号は他文書から `P1` のように参照される。P1・P2・P3・P5・P6 の文言は確定デザインのヘッダ宣言を取り込んだもの。P4・P7 は旧仕様から維持。

- **P1: 忠実性。** 翻訳もチャット回答も原文を正とし、根拠(セクション/式/図のアンカーチップ)を必ず示す。訳文・要約・図解はすべて派生物であり、対応する原文箇所へ 1 アクションで戻れること。AI 生成物には「AI生成」「論文外の知識」「推測」ラベルを付け、本文由来と区別する。ユーザーの信頼はここで決まる。
- **P2: 1アクション・段階的提示。** 取り込みは URL 1 つ(ブラウザ拡張の「保存」)。追加直後にカードが現れ、読める部分から段階的に翻訳が進む(書誌 → アブスト訳 → 本文)。「全部できるまで待たせる」設計を禁止する。
- **P3: 黙って壊れない。** 翻訳失敗ブロックは原文+理由を表示する。位置を失った注釈は「未配置」として残す。勝手に消さない・勝手に状態を変えない。品質と失敗は品質レベルバッジ(A/B)とブロック単位の原文フォールバックで可視化し、常に再試行の手段を提供する。壊れた訳文を黙って見せることは、失敗を表示するより悪い。
- **P4: 読んだ痕跡はすべて位置に紐づき、あとから検索できる。** ハイライト・メモ・チャットは論文上のアンカーを持ち、「あの論文のあの話」を数ヶ月後に想起できることを保証する。横断検索(4e)はヒット源(本文/訳文/メモ/チャット/記事)を明示する。
- **P5: ロックインしない。** メモ/注釈/チャットは Markdown、書誌は BibTeX/CSV、全量 JSON でいつでも持ち出せる。データはユーザーのもの。
- **P6: 読書の邪魔をしない・管理を強要しない。** 落ち着いた配色・少ない色味・控えめな統計。AI 機能は呼んだときだけ現れ、本文閲覧中に自発的に割り込まない。進捗は読む行動から育つ: ステータスの自動変更はせず提案のみ、連続記録などのゲーミフィケーションはしない。
- **P7: 著作権・ライセンスを設計で守る。** 全文翻訳は取り込んだ本人の閲覧に限る。共有できるのはコレクション共有ページ(4c)の「書誌+要約+共有者が許可したメモのみ」。図表の転載(記事モード内)は原論文ライセンスが許す場合に限り、クレジットを自動付記する(詳細: [09-nonfunctional.md](09-nonfunctional.md))。

## 3. 対象ユーザー(ペルソナ)

### U1: 企業研究者(主要ペルソナ)
機械学習・ロボティクス等の分野で週 5〜20 本の論文に目を通し、うち 2〜3 本を深く読む。英語は読めるが日本語の 2〜3 倍時間がかかる。困りごと: トリアージに時間を使いすぎる/読んだ論文の内容を数ヶ月後に思い出せない/数式まわりを英語で精読するのがつらい。

### U2: 実装エンジニア
論文の手法を再実装する。必要なのは思想よりも「損失関数の各項、ハイパーパラメータ、学習手順、実装上の落とし穴」。図・式・実験設定テーブルに素早くアクセスしたい。論文にひも付いた公式実装(GitHub)や解説動画にも同じ場所からアクセスしたい(リソースタブ 5a、[12-resources.md](12-resources.md))。

### U3: 学生・英語学習者
輪読会で担当論文を読む。専門用語の定訳が分からない。訳文だけ読むと英語力がつかないため、対訳で読みつつ語彙を貯めたい。単語・コロケーション・イディオムを論文の文脈ごと保存し、あとから復習したい(語彙帳 4d、[11-vocabulary.md](11-vocabulary.md))。

### U4: 輪読会・チームのオーガナイザ
読むべき論文リストをチームで共有し、締切と担当を管理したい。メンバーの環境を選ばない共有手段(URL)が必要。共有相手にアカウント作成を強要したくない。

## 4. 主要シナリオ

機能仕様は以下のシナリオを満たすことを第一の受け入れ条件とする。各シナリオ末尾の「⇒」は、そのシナリオが要求する仕様上の要点。

### S1: 朝のトリアージ(10分・デスクトップブラウザ+拡張)
Slack で流れてきた arXiv リンク 3 本を、仕事開始前にデスクトップブラウザで処理する。abs ページを開くとツールバーの「A」アイコンからポップアップ「Alineaに保存」(3a)が開き、書誌と「✓ LaTeX ソースあり — 品質レベル A 見込み」が保存前に表示される。1 本目はステータス「すぐ読む」+タグ提案を採用して「保存 ⏎」。2 本目は「読む予定」+コレクション「輪読会 2026-07」へ。3 本目はポップアップが「既にライブラリにあります」と検知し、重複を作らず「続きから開く ↗」を出す。保存直後は同じポップアップ内でパイプライン進捗(書誌✓→構造化✓→翻訳中%)が見え、読める部分からすぐ開ける。移動中のスマホでは、届いた通知からライブラリを開いてステータスを変えるだけに留める。
⇒ 拡張が唯一の取り込み経路(アプリ内「+追加」は持たない)/ 保存前の書誌プレビューと品質見込み / 重複検知 / 取り込み時のステータス・タグ・コレクション指定 / 段階的パイプライン進捗 / モバイルは閲覧・ステータス変更のみ([08-extension.md](08-extension.md), [02-ingest.md](02-ingest.md), [06-library.md](06-library.md))

### S2: 深い読解(90分・デスクトップ)
「すぐ読む」から論文を開き、対訳モード(1a)で読む。式(5)の意味が分からず、数式ブロックの「✦この式を説明」で根拠チップ(段落粒度)つきの説明が返る。本文中の「図3」参照をクリックするとその場にポップオーバーで図が出る(1c。スクロールで流れを失わない)。"rectified flow" の訳語が気に入らないので原語のままに固定すると、影響する段落だけ再翻訳される。気になる表現は選択メニューの「語彙に追加」で文脈ごと語彙帳へ送る。実装が気になったらリソースタブで自動検出された「公式実装」を開く。途中で離脱すると位置が保存され、「読んでいる」への変更が通知で提案される(自動変更はしない)。
⇒ 対訳ビュー / 選択質問と根拠表示 / 参照ポップオーバー / 用語固定と部分再翻訳 / 語彙への追加 / リソース参照 / 位置保存とステータス提案([03-translation.md](03-translation.md), [04-viewer.md](04-viewer.md), [05-chat.md](05-chat.md), [11-vocabulary.md](11-vocabulary.md), [12-resources.md](12-resources.md))

### S3: 読了と記録(5分)
読み終えて「読んだ」にすると読了フローのモーダル(1g)が出る。読了日と累計読書時間は自動記録の表示のみ。理解度 1〜5(例:「4/5 — だいたい追えた」)・重要度(低/中/高)・ひとことメモ(「何に使えるか」)を軽く入力する(「すべてスキップ」可)。「✦要約をメモに保存」で要約をメモに昇格し、「記事モードで読み返す→」の導線が次の読み返しにつながる。
⇒ 読了フロー / チャット・要約→メモ昇格 / 読書時間の自動記録([05-chat.md](05-chat.md), [06-library.md](06-library.md))

### S4: 輪読会共有(30分)
論文をコレクション「輪読会 2026-07」に入れ、順序・締切・担当を設定する(4b)。共有リンク(`alinea.app/c/{token}`)を発行し、「共有ページにメモを含める」を有効にしてチームに送る。受け手はアカウント不要で共有ページ(4c)を開き、「共有されたコレクション — 閲覧専用」の一覧(書誌+✦要約+「共有者のメモ」)を見る。ページは noindex で、未登録者には「Alineaをはじめる」CTA が出る。各メンバーは自分のライブラリで論文を読み、読了後は各自の記事モード(1h)で読み返して発表準備をする。発表用にはデスクトップの「✦ ツール」から論文スライド(PPTX)を生成でき、仕上げた記事は任意で公開スナップショットとして共有・コメント受付もできる。
⇒ コレクションの順序・締切・担当 / トークン URL 共有(閲覧専用・アカウント不要・noindex)/ メモの共有可否制御 / 各自の記事モード読み返し([06-library.md](06-library.md), [07-figures-and-articles.md](07-figures-and-articles.md), [09-nonfunctional.md](09-nonfunctional.md))

### S5: 想起(2分)
3ヶ月後、「EMA teacher を使ったオンライン蒸留の論文はどれだったか」を横断検索(1e ドロップダウン → 4e)で探す。本文・訳文・メモ・チャット・記事を横断してヒットし、ヒット源バッジで区別され、日本語で検索すれば訳文に、英語で検索すれば原文に当たる。当時のハイライト位置へジャンプできる。
⇒ 原文・訳文・注釈・チャット履歴・記事の横断全文検索(日英クロス・ヒット源明示)/ アンカーへのジャンプ([06-library.md](06-library.md))

### S6: すきま時間の語彙復習(5分)
語彙帳(4d)を開くと、忘却曲線に基づき「復習期」の語彙エントリが提示される。保存時に文脈から自動生成された語義・語源・解釈のしかた・覚えるコツを読み、「まだあやしい」/「✓覚えた」で自己評価すると次の復習日が更新される。「原文で見る→」で保存元のセンテンスに戻れる。
⇒ 文脈つき語彙保存 / AI 自動生成の学習カード / SRS 復習 / 原文アンカーへの往復([11-vocabulary.md](11-vocabulary.md))

## 5. スコープ

### 対象
- 原文言語: 英語。訳出言語: 日本語。
- 入力経路: **ブラウザ拡張(Chrome / Edge)に一本化**(3a、[08-extension.md](08-extension.md))。arXiv abs ページの URL 送信(最優先。「URL のみを送信します — 取得・解析はサーバーで実行」)と、一般ページで表示中の PDF の明示送信(「このタブの PDF を送信」。「private 論文として保存され、共有されません」)。アプリ内の「+追加」ボタン/取り込みダイアログは持たない。対応サイト: ACL Anthology / OpenReview / PubMed・PMC(JATS)/ Hugging Face 経由の arXiv 同定(実装済み)。出版社ページ(IEEE/ACM)は追加候補([02-ingest.md](02-ingest.md))。
- 品質レベル: **A(LaTeX から完全構造化)/ B(PDF 由来)の 2 段階**。バッジで常時明示([02-ingest.md](02-ingest.md))。
- ビューアの表示モード: **訳文 / 対訳 / 原文 / PDF / 記事** の 5 つ。サイドパネルは排他 6 タブ(チャット / メモ / 注釈 / 図表 / リソース / 情報)([04-viewer.md](04-viewer.md))。アウトプット: 論文単位スタンドアロン HTML/PDF エクスポート、記事公開、**論文からスライド(PPTX)生成**(デスクトップ「✦ ツール」、[07-figures-and-articles.md](07-figures-and-articles.md))。
- プラットフォーム: デスクトップ Web(基準ビューポート 1440×900、ダークモード対応)+ Chrome / Edge 拡張。**モバイルは閲覧とステータス変更のみ**(取り込み・記事再生成などの操作系は v1 スコープ外)。
  - 決定: 旧仕様の「モバイルからの取り込み」は、取り込みが拡張一本化されたことに伴い v1 から外す(理由: モバイルブラウザは拡張非対応であり、代替のアプリ内取り込み UI は確定デザインに存在しない)。

### 実装済み(旧・非目標から昇格)
- **記事の公開・限定公開・コメント(M3 実装済み)**: 記事は第一にビューアの表示モードだが、任意でサニタイズ済みスナップショットを slug URL(`/p/{slug}`、unlisted=noindex / public=索引可)で公開し、ブロック単位のコメントを受け付けられる([07-figures-and-articles.md](07-figures-and-articles.md))。P7 整合として公開版には原文引用本文・訳文・メモ・チャット・原論文図を含めない。旧仕様のブロックエディタ・複数記事・公開レイアウト・OGP は記事モード仕様に置き換え済み(記事は論文×ユーザーで 1 つ)。

### 非目標(明示的にやらないこと)
- **論文の発見・レコメンド**: 検索エンジンや新着フィードは作らない。取り込みの起点は常にユーザー。(arXiv 新着ウォッチは将来の検討事項に留める)
- **引用管理・執筆支援**: BibTeX の体系的管理、Word/LaTeX 執筆連携は対象外。BibTeX エクスポートのみ提供。
- **英語以外の原文**(中国語論文等): 当面対象外。
- **リアルタイム共同編集**: メモの同時編集はしない。共有は「閲覧」と「リンク共有」まで。
- **公開ディスカッション基盤**: 論文ページへの公開コメント(alphaXiv 型)はしない。
- **汎用ドキュメント翻訳**: 論文以外の文書(仕様書・記事など)は対象外。
- **アプリ内取り込み UI**: URL 貼り付け欄・PDF ドラッグ&ドロップは持たない(拡張へ集約)。

## 6. プロダクト用語

| 用語 | 意味 |
|---|---|
| 論文 (Paper) | サービス全体で一意な論文実体。arXiv ID / DOI 等で正規化される |
| ライブラリ項目 (LibraryItem) | ユーザー × 論文。ステータス・タグ・優先度・締切・メモ等の個人状態を持つ |
| 読書ステータス | 6 値: 読む予定 / すぐ読む / 読んでいる / 読んだ / あとで再読 / 保留([06-library.md](06-library.md)) |
| 構造化ドキュメント | 入力形式によらない論文本文の共通中間表現([01-domain-model.md](01-domain-model.md)) |
| ブロック | 構造化ドキュメントの最小単位(段落・図・表・数式など)。安定 ID を持つ |
| 翻訳セット | ドキュメント × スタイル × 用語集に対する訳文一式 |
| アンカー | 論文上の位置参照(ブロック ID + 文字オフセット)。注釈・チャット根拠・語彙・検索ジャンプが使う |
| 注釈 (Annotation) | ハイライト・コメント・ブックマークの総称。4 色(重要/疑問/アイデア/用語) |
| 記事モード | ビューアの表示モードの 1 つ(1h)。訳文・メモ・チャット履歴を素材に AI がブログ風に再構成した読み返しビュー。ブロック単位の「✦書き直し指示」(ホバーツールバー)で再生成できる。独立エディタではない([07-figures-and-articles.md](07-figures-and-articles.md)) |
| 概要図 | 記事モード冒頭の全体概要図。課題→提案→結果の 3 カードフロー。構造化図データから SVG を決定的にレンダリングし、版管理・指示つき再生成・SVG ダウンロードを持つ |
| 解説図 | 記事内の詳細解説図・挿絵。画像生成 API(プロバイダ切替可)によるラスター画像([07-figures-and-articles.md](07-figures-and-articles.md)) |
| リソース (ResourceLink) | 論文に添付する外部学習材料(GitHub 実装 / 発表動画 / スライド / 解説記事)。URL 貼り付けで種別自動判定、arXiv 由来の公式実装は自動検出(5a、[12-resources.md](12-resources.md)) |
| 語彙エントリ (VocabEntry) | 英語学習のために文脈ごと保存する 単語 / コロケーション / イディオム。語義・語源・解釈のしかた・覚えるコツを持ち、忘却曲線(SRS)で復習する(4d、[11-vocabulary.md](11-vocabulary.md))。訳語統一の用語集とは別物 |
| コレクション | 手動キュレーションの順序付き論文リスト(プロジェクト・輪読会用)。締切・担当・共有リンクを持つ |
| 共有ページ | コレクションのトークン URL 閲覧ビュー(4c)。アカウント不要・閲覧専用・noindex |
| 品質レベル | 取り込み結果の構造化度合い。A = LaTeX から完全構造化 / B = PDF 由来の 2 段階([02-ingest.md](02-ingest.md)) |

## 7. 未決事項(プロダクトレベル)

実装を始める前にオーナーの判断が必要な項目。暫定案で仕様は書き進める。Q2・Q5 は 2026-07 のデザイン確定・ユーザー決定により更新済み。

| # | 論点 | 暫定案(仕様が仮定する姿) | 代替案 |
|---|---|---|---|
| Q1 | 提供形態 | 個人開発のホスト型サービス(クローズドβ→公開)。コードは OSS 化余地を残す | 完全 OSS セルフホスト / 社内ツール |
| Q2 | LLM コスト負担 | **決定済み**: 運営キー(クォータ付き)+ 自分の API キー持ち込み(BYOK)。テキスト生成は GPT / Claude / Gemini / DeepSeek、画像生成は GPT / Gemini / Grok を用途別に切替可能(設定 4f、[09-nonfunctional.md](09-nonfunctional.md)) | 有料プラン / 完全 BYOK |
