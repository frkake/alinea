# 03. 日本語訳の生成

> 翻訳品質はこのプロダクトの生命線。「速く・安く」より「原文に忠実で・数式や参照が壊れず・用語が一貫している」を優先する(P1, P3)。訳文は常に「✦ AI翻訳」であることを明示し、疑わしい訳は 1 アクションで原文と突き合わせ・再翻訳できる。

## 1. 翻訳の原則(優先順位つき)

1. **忠実性**: 原文の主張・限定・ニュアンス(may/might/we hypothesize 等)を保つ。要約・省略・補足をしない。
2. **構造保持**: 段落と原文は 1:1 対応。文の分割・結合は許すが、段落をまたぐ再構成・順序入替は禁止(対訳モードの段落単位 2 カラム表示(1a)と、対訳ポップの ¶ 単位対応(1b)の前提)。
3. **記号の完全保持**: 数式・引用・参照・URL・コードは 1 文字も変えない(§4 プレースホルダで機械的に保証)。
4. **用語の一貫性**: 同一論文内で同一用語は同一訳(§7)。
5. **AI 生成の明示**: 訳文カラムには「✦ AI翻訳」を常時表示する(1a)。翻訳を人間の訳と誤認させない(P1)。
6. **自然さ**: 上記を満たす範囲で、学術書として読みやすい「だ・である」調の日本語。

## 2. 翻訳対象と既定設定

| 対象 | 扱い |
|---|---|
| 本文段落・リスト・定理環境・脚注 | 翻訳する |
| 見出し | 翻訳し、**原題を併記**(「1 はじめに — Introduction」(1b))。原題は参照・記憶の手がかりになる |
| 図表キャプション | 翻訳する |
| アブストラクト | 取り込み時に最優先で翻訳(+3行要約を別途生成。要約は [05-chat.md](05-chat.md)) |
| 数式・コード・アルゴリズム擬似コード | 翻訳しない(コメント行のみ任意翻訳) |
| 参考文献リスト | 翻訳しない(書誌情報のため)。目次では淡色表示し、翻訳進捗の分母に含めない(1a) |
| 著者名・所属・モデル名・データセット名等の固有名詞 | 原語のまま |
| 付録(Appendix) | **既定では自動翻訳しない**。「開いたとき・ボタンでオンデマンド翻訳(コスト対策)」(4f)。多くの読者は付録まで読まない |
| 表のセル内テキスト | 既定では翻訳しない。「数値・記号が主のため。表単位の「この表を翻訳」は常に利用可」(4f) |

上記の「既定」は設定画面 (4f) の「翻訳」カテゴリでユーザーが変更できる。設定項目と文言はデザイン逐語に一致させる:

| 設定項目(4f 逐語) | 型 | 既定 | 説明文(4f 逐語) |
|---|---|---|---|
| 「既定の翻訳スタイル」 | 2択ラジオカード | 「● 自然訳(既定)」 | 自然訳=「こなれた学術日本語。取り込み時に自動生成」/「○ 直訳」=「原文の語順・構文を写像。文単位で対応を追える。初回切替時にオンデマンド生成」 |
| 「付録(Appendix)を自動翻訳しない」 | トグル | ON | 「開いたとき・ボタンでオンデマンド翻訳(コスト対策)」 |
| 「表のセル内テキストを翻訳しない」 | トグル | ON | 「数値・記号が主のため。表単位の「この表を翻訳」は常に利用可」 |
| 「30 ページ超の論文はセクション選択を提案」 | トグル | ON | 「全文翻訳の前に翻訳対象を選べます(既定は全選択)」 |

- スタイル選択カードの補足注記(4f 逐語): 「切替は表示の切替であり、原文表示とは独立です。文体は「だ・である」調に固定」。文体の設定項目は存在しない。
- 「この表を翻訳」は設定値に関わらず常に利用可(4f)。設定はあくまで「自動翻訳の範囲」を決める。

## 3. 翻訳の単位と実行

- 単位: **ブロック単位**(段落・キャプション等)。ただしプロンプトには前後ブロックとセクション見出し、用語スナップショットを文脈として与える(文脈なし翻訳は代名詞・省略の誤訳を生む)。
- 実行順: アブストラクト → イントロダクション → 以降セクション順(P2: 書誌 → アブスト訳 → 本文と段階的に読めるようになる)。ユーザーが開いているセクションを最優先に繰り上げる([02-ingest.md](02-ingest.md))。
- 訳文は TranslationUnit として保存し、`source_hash` で原文との対応を検証する([01-domain-model.md](01-domain-model.md))。
- 翻訳完了はアプリ内通知(通知アイコン◷)で知らせる(「翻訳完了」→「読み始める →」。通知仕様は [06-library.md](06-library.md))。取り込み直後の進捗はブラウザ拡張のパイプライン表示「書誌✓→構造化✓→翻訳中%」にも出る([08-extension.md](08-extension.md))。

## 4. プレースホルダプロトコル(記号保護)

翻訳 LLM には `text` 以外のインラインをトークン化して渡し、復元時に機械検証する。

```
原文:  We train with ⟦CIT:ref-12⟧ using the loss in ⟦REF:eq-5⟧, where ⟦MATH:m-3⟧ denotes ...
訳文:  ⟦REF:eq-5⟧ の損失を用いて ⟦CIT:ref-12⟧ に従い学習する。ここで ⟦MATH:m-3⟧ は ... を表す。
```

- 検証: 出力に全トークンが **ちょうど 1 回ずつ** 現れること。失敗時は最大 2 回再試行し、それでも失敗したら **そのブロックは原文のまま表示**+`quality_flags: placeholder_mismatch` を付けて「訳せなかった段落」として一覧化する(P3)。壊れた訳を見せるくらいなら原文を見せる。
- トークンの順序移動は許可する(日本語の語順のため)。削除・複製・改変は不合格。
- 相互参照トークンは復元後、ビューアの点線下線リンク(「図2」「式(5)」)・文献参照(「[12]」)として描画される(1a)。プロトコルが守られる限り参照リンクは翻訳で壊れない。

## 5. 翻訳スタイル

| スタイル | 目的 | 生成タイミング |
|---|---|---|
| 自然訳(既定) | 通常の読解。「こなれた学術日本語」(4f) | 「取り込み時に自動生成」(4f) |
| 直訳 | 英語学習・厳密読解。「原文の語順・構文を写像。文単位で対応を追える」(4f) | **オンデマンド**。「初回切替時にオンデマンド生成」(4f)。全ユーザーに常時 2 倍のコストをかけない |
| (将来)やさしい訳 | 初学者向け。専門用語に短い注を付す | M3([10-roadmap.md](10-roadmap.md)) |

- スタイル切替はビューアヘッダのドロップダウン「スタイル: 自然訳 ▾」(1a)(1b)。既定スタイルは設定 (4f) の「既定の翻訳スタイル」で選ぶ。
- 「切替は表示の切替であり、原文表示とは独立です」(4f)。原文モード・対訳モードの有無とスタイル選択は直交する。
- 決定: 直訳のオンデマンド生成は、切替時に表示中のセクションから優先して生成し、以降は自然訳と同じキュー(§3)で進める。一度生成した直訳はキャッシュし、以後の切替は即時表示。理由: 「初回切替時に生成」というデザイン文言と、切替のたびに課金しないことの両立。
- 「分野別の翻訳スタイル調整」は独立スタイルとしては持たず、**用語集(§7)+分野プロファイル**(arXiv カテゴリから選ばれるプロンプト前提: 例 cs.LG なら ML 慣行の訳語感覚)として実現する。スタイル軸を増やすとキャッシュ(§8)が分裂するため。

### 訳文の文体規定(プロンプト仕様に含める)

- 「だ・である」調に固定(4f)。体言止め多用禁止。
- 慣用のカタカナ語は無理に訳さない(attention → アテンション、fine-tuning → ファインチューニング)。定訳のある語は定訳(neural network → ニューラルネットワーク、generalization → 汎化)。
- 初出の頭字語は「日本語訳(English, ABBR)」形式(例: 「大規模言語モデル(Large Language Model, LLM)」)。以降は略語のみ。
- 用語ハイライト対象の初出訳は原語併記(例: 「整流フロー(rectified flow)」(1a)(1b))。
- 良い/悪い訳の対訳例集を仕様アセットとして整備し、プロンプトとリグレッションテストの両方に使う。

## 6. ビューアでの翻訳の提示

ビューア全体の仕様は [04-viewer.md](04-viewer.md)。ここでは翻訳状態がどう見えるかだけ定める。

### 6.1 翻訳進捗と未翻訳の明示(1a)

- 目次ヘッダに論文全体の翻訳進捗率を表示する(例「翻訳 96%」)。分母は自動翻訳対象ブロック(参考文献・付録・表セルなど対象外は含めない。理由: 既定で訳さないものを分母に入れると 100% に到達せず、進捗表示が嘘になる)。
- 目次の各節に翻訳済みチェック「✓」を表示。未翻訳・対象外の項目(「参考文献」)は淡色。
- 未翻訳の付録は目次末尾に破線ボックスで明示する(逐語): 「付録 A 証明 — 未翻訳」+「開くと翻訳します(オンデマンド)」。目次から付録を開くとオンデマンド翻訳が走る(§2 の設定と連動)。

### 6.2 見出しの原題併記(1a)(1b)

- 節見出しは「訳題 — 原題」形式。例: 「1 はじめに — Introduction」「2.1 整流フロー — Rectified Flow」「アブストラクト — Abstract」。原題部分は英文書体(Source Serif 4 イタリック)・淡色。

### 6.3 「✦ AI翻訳」表示と対訳ポップ(1a)(1b)

- 対訳モードのカラムヘッダは「原文 — ENGLISH」/「訳文 — 自然訳 ✦ AI翻訳」+「段落対応 ⇄」(1a)。訳文が AI 生成であることをカラム単位で常時明示する。
- 訳文モードでは段落ホバーで「対」ボタンが出現し、段落直下に原文をインライン展開する(対訳ポップ)。キーボード「t で開閉」。ポップには段落位置(例「¶2 / 1 Introduction」)を表示する(1b)。
- 対訳ポップのフッターが再翻訳への導線(1b 逐語): 「訳がおかしい?」→「再翻訳」「指示つき再翻訳」。訳を疑ったらまず原文を突き合わせ、そのまま同じ場所から再翻訳に進める(§9)。

## 7. 用語管理(3層用語集)

適用優先度: **論文ローカル > ユーザー > グローバル既定**。

- **グローバル既定**: 運営が整備する分野別の定訳シード(ML/CV/NLP/ロボティクスから開始)。
- **ユーザー用語集**: ユーザーが育てる自分の訳語。エントリは「原語 / 訳語 / 訳し方(訳す・原語のまま・併記)」。
- **論文ローカル用語集**: 取り込み時に論文から頭字語・新規手法名・頻出専門用語を自動抽出し、訳語案とともに「この論文の用語」パネルに提示する。ユーザーはここで訳語を確定・修正できる。

挙動:

- 翻訳時には適用される用語のスナップショットを凍結し、TranslationSet に記録する(再現性)。
- **訳語を変更したら、その用語を含むブロックだけを再翻訳する**。全文再翻訳は求めない。影響ブロック数を事前に表示する(「12 段落を再翻訳します」)。
- 用語パネルからワンクリックで「ユーザー用語集に昇格」できる(次の論文から効く)。

注意: この用語集は**訳語統一のための内部機構**であり、英語学習機能の「語彙帳」(4d) とは別物([11-vocabulary.md](11-vocabulary.md))。選択メニューの「語彙に追加」(1b) は語彙帳に入り、用語集には影響しない。

## 8. キャッシュと共有

- public 論文 × 自然訳 × グローバル既定用語のみ、の組合せは **サービス全体で共有**する(`scope: shared`)。2 人目以降は翻訳待ちゼロ・コストゼロ。
- 直訳も同条件(public × グローバル既定用語のみ)なら shared キャッシュに入る。誰かが一度オンデマンド生成すれば以降のユーザーは待たない。
- ユーザー用語集・論文ローカル用語集を適用した時点で、そのユーザー専用のフォーク(`scope: personal`)を作る。フォークは共有版との差分(影響ブロック)のみ保持する。
- private 論文(アップロード PDF・出版社ページ由来)は常に personal。共有プールに入れない([09-nonfunctional.md](09-nonfunctional.md))。

## 9. 再翻訳・手動修正・差分

- ブロック単位の操作: 「再翻訳」/「指示つき再翻訳」(例: もっと簡潔に、専門用語を残して)/「手動編集」。入口は対訳ポップのフッター「訳がおかしい?」(1b、§6.3)。
- 再翻訳は §11 の方針に従い、通常翻訳より上位のモデルにエスカレーションして実行する(同じモデルで引き直しても同じ誤りを繰り返しやすい)。
- 手動編集したブロックは `state: edited` となり、以後の自動再翻訳で **上書きしない**(明示的に「編集を破棄して再翻訳」した場合のみ)。
- 再翻訳時は新旧の差分をインライン表示し、採用/破棄を選べる。
- 「訳がおかしい?」の押下は品質テレメトリとして記録する(どの論文・どのブロックで訳が疑われたか)。プロンプト・用語集改善の材料にする。

## 10. 自動品質検査

各 TranslationUnit に対して機械チェックを行い、`quality_flags` を付与する。フラグ付きブロックはビューアで下線表示し、対訳ポップを促す。

- プレースホルダ検証(§4)
- 数値・単位の一致(原文中の数値が訳文に同数出現するか)
- 長さ逸脱(原文比で極端に短い/長い → 省略・幻覚の疑い)
- 用語スナップショットからの逸脱
- 未訳(英文がそのまま残っている)検出

## 11. コスト制御とモデル選択

- 付録・参考文献・表セルを既定で訳さない(§2、4f のトグルで変更可)。
- 「30 ページ超の論文はセクション選択を提案」(4f): 全文翻訳の前に翻訳対象セクションを選べる(既定は全選択)。提案であって強制ではない(P6)。
- 直訳はオンデマンド生成(§5)。使われない訳を先回りで作らない。
- **モデルとプロバイダ**: テキスト生成モデルは用途別に OpenAI GPT / Anthropic Claude / Google Gemini / DeepSeek をプロバイダ切替できる(運営既定+ユーザー設定+BYOK)。本仕様が定めるのは方針のみ:
  - **翻訳 = 安価な大量処理向けモデル**(論文 1 本で数百ブロックを訳すため、単価が支配的)。
  - **再翻訳・「指示つき再翻訳」= 上位モデルへエスカレーション**(困った時の脱出口。件数が少ないので単価より品質)。
  - 翻訳用のプロバイダ・モデル選択と BYOK の設定 UI は、設定 (4f) の既存カテゴリ「翻訳」内に追加する(デザインの 8 カテゴリ構成は変えない。チャット用は「チャット」、キー管理は「アカウント」)。
  - 具体のルーティング表・モデル ID・フォールバック連鎖は [09-nonfunctional.md](09-nonfunctional.md) と実装計画に委ねる。
- ユーザー別クォータと BYOK は [00-product.md](00-product.md)(Q2)に従う。

## 12. 受け入れ基準

- [ ] プレースホルダ保持率(検証通過率)99.9% 以上。不合格ブロックが訳文として表示されることがない
- [ ] 同一論文内で用語スナップショット中の用語の訳ゆれゼロ
- [ ] 見出しに原題が併記される(「1 はじめに — Introduction」形式)
- [ ] 訳文カラム/訳文モードに「✦ AI翻訳」表示がある
- [ ] 目次に翻訳進捗率と節ごとの ✓ が表示され、未翻訳付録は「開くと翻訳します(オンデマンド)」と明示される
- [ ] 設定 (4f) の 4 項目(既定スタイル/付録/表セル/30 ページ超提案)が翻訳挙動に反映される
- [ ] 直訳が初回切替時にオンデマンド生成され、2 回目以降の切替は即時
- [ ] 対訳ポップの「訳がおかしい?」から再翻訳・指示つき再翻訳に到達できる
- [ ] 再翻訳が通常翻訳より上位のモデルで実行される
- [ ] 訳語変更 → 影響ブロックのみ再翻訳、が 1 分以内に反映される
- [ ] 手動編集ブロックが再翻訳で上書きされない
- [ ] 2 人目のユーザーが同一 public 論文を追加したとき、翻訳待ちなしで読める
