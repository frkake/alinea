# 09-screens/mobile. モバイル縮退レイアウト実装仕様(M1)

> 対象読者と前提: 本書は「訳読 / YAKUDOKU — 論文読解ワークベンチ」の apps/web(Next.js 15 App Router + React 19 + TypeScript 5 + Tailwind CSS v4)実装者向けに、モバイル縮退レイアウトの実装仕様を確定させる計画書である。機能仕様は docs/00 §5(モバイルは閲覧とステータス変更のみ)・docs/04 §13(訳文モード+タップ対訳・ボトムシート・目次ドロワー)・docs/09 §6 を正とする。共通コンポーネントは plans/08-design-system.md、ビューア骨格は plans/09-screens/viewer-shell.md、ライブラリ/シェルは 4a・1d・1e の各計画書を参照し、本書では再定義しない。**モバイル縮退の所有者は本書である**(ブレークポイント・対象範囲・実装方式・受け入れ基準の正)。本書に書かれた値・識別子が実装の正であり、独自の解釈を禁止する。実装マイルストーンは **M1**。

## 1. 対象と非対象(docs/00 §5 のスコープを実装に確定)

### 1.1 対象(モバイルで動作を保証する機能)

| 画面 | モバイルでの提供内容 |
|---|---|
| ビューア `/papers/[itemId]` | **訳文モードのみ**。段落タップで対訳ポップ(§4.4)。目次ドロワー・閲覧専用ボトムシート(§4) |
| ライブラリ `/library` | **カードビューのみ**(テーブルビューは提供しない)。1 カラム化(§5.1) |
| ダッシュボード `/dashboard` | 縦積み 1 カラム化(§5.2)。閲覧のみ |
| ステータス変更 | ビューアヘッダのステータスピル+ライブラリカード上のステータスピルから 6 値(planned / up_next / reading / done / reread / on_hold)を変更可(§5.3) |

### 1.2 非対象(モバイルでは UI を非表示にする操作系)

次の操作系 UI はモバイル縮退時に**条件レンダリングで DOM から除外する**(`display:none` ではなく非描画。決定。理由: フォーカス到達・誤操作・転送量の排除)。

1. 取り込み(そもそも拡張機能のみ。モバイル向け代替 UI は作らない。docs/00 §5 の決定どおり)
2. 注釈作成(選択メニュー・ハイライト・コメント入力。既存注釈の**閲覧**はボトムシートで可)
3. チャット送信(入力欄・送信ボタン。チャット**履歴の閲覧**はボトムシートで可)
4. 語彙追加
5. 記事再生成(`ArticleRegenerateButton` は非描画)
6. 一括操作(ライブラリの複数選択・バルクバー)
7. 設定の一部: 変更系フォーム(プロフィール編集・API キー・エクスポート実行等)は非描画。設定値の**参照は可**

上記以外の操作系(再翻訳・用語固定・コレクション編集・保存フィルタ編集等)もすべて非表示とする。モバイルで操作可能なのは「閲覧に伴うナビゲーション+ステータス変更」だけである(決定)。

## 2. ブレークポイント(決定)

- **< 768px: モバイル縮退レイアウト**(本書)。判定は CSS メディアクエリ `@media (max-width: 767px)` と、JS 側は `useMediaQuery('(max-width: 767px)')` フック(`apps/web/src/hooks/useMediaQuery.ts`。`matchMedia` + `change` 購読)の 2 系統で行い、値は定数 `MOBILE_MAX_WIDTH = 767` を `packages/tokens` に置く。
- **768〜1199px: デスクトップ縮小**。plans/08 §7.2 の `MIN_APP_WIDTH = 1200`(`body { min-width: 1200px }`)をそのまま適用し、水平スクロールを許容する。中間レイアウトは作らない(決定。plans/08 §7.2 の既存規則と整合)。
- **≥ 1200px: デスクトップ**(plans/08 §7.2)。
- `min-width: 1200px` との関係(決定): `body` の min-width 指定は `@media (min-width: 768px)` 内に移動する。< 768px では `min-width: 0` とし、モバイル縮退レイアウトが 100vw に収まる。plans/08 §7.2 の記述はこの条件付けを前提に読む(モバイルは同節の「対象外」とされており、本書が所有する)。

## 3. 基準ビューポート

- **390×844**(iPhone 12/13/14 系)。E2E テスト PW-22(plans/12 §PW-22)はこのビューポートで実行する。
- 検証環境: iOS Safari / Android Chrome(docs/09 §6)。開発時の一次確認は Chromium の device emulation 390×844 でよい(決定)。

## 4. ビューア(`/papers/[itemId]`)のモバイル縮退

### 4.1 モード固定

- モバイルでは **`mode` を `translation` に固定**する(決定)。`?mode=` クエリに他の値(parallel / source / pdf / article)が来た場合も `TranslationPane` を描画し、URL は書き換えない(デスクトップに戻れば元モードで開けるため。決定)。
- ヘッダのモード切替タブは非描画とし、代わりに**バッジ**「訳文」(`Badge` 相当。h18px、font 10px/600、bg `var(--pr-acc-s)`、color `var(--pr-a)`)を表示する(決定)。

### 4.2 ヘッダ縮退

- `ViewerHeader`(viewer-shell §4)は h52px のまま、表示要素を **戻るリンク(← `/library`)/ 論文タイトル(1 行省略)/ ステータスピル / 訳文バッジ** の 4 つに縮退する(決定)。スタイルセレクタ・論文内検索・オーバーフローメニュー・パネル開閉ボタンは非描画。
- ステータスピルは viewer-shell §4.2-4 と同一挙動(Popover 6 値、`PATCH /api/library-items/{id} { status }` 楽観更新、`done` 選択時は読了フローモーダル 1g を経由)。ただしモバイルでは Popover を画面幅 −32px・placement `bottom` で出す(決定)。

### 4.3 目次 = ドロワー

- `TocRail` / `TocPane` は非描画。ヘッダ左端の戻るリンク右隣に目次ボタン(≡ アイコン 20px)を置き、タップで**左からオーバーレイするドロワー**(width 280px、height 100dvh、bg `var(--pr-bg-panel)`、右 border 1px `var(--pr-border)`、backdrop rgba(0,0,0,0.4)、transition transform 200ms)を開く(決定)。中身は `TocPane` の行コンポーネント(`TocRow`)を再利用。行タップでその節へスクロールし、ドロワーを閉じる。backdrop タップ / Esc で閉じる。
- コンポーネント: `apps/web/src/components/viewer/toc/TocDrawer.tsx`(新規。本書所有)。

### 4.4 本文とタップ対訳

- 本文は `TranslationPane`(1b 所有)を 1 カラム・padding 左右 16px・本文幅 100% で描画(720px 固定カラムは解除。決定)。
- **段落タップで対訳ポップ**: デスクトップの「ホバーで対訳アイコン → クリックで展開」(1b)に代えて、モバイルでは**段落本体のタップでインライン対訳ポップ(`ParallelPop`)を開閉**する(決定)。同時に開けるのは 1 段落(1b の `openPopBlockId` 排他をそのまま使用)。ポップ内の再翻訳フッタ(`RetranslateFooter`)は操作系のため非描画(決定)。
- テキスト選択メニュー(`SelectionMenu`)は表示しない(注釈作成・AI 質問・語彙追加が非対象のため。決定)。
- ブロック深リンク `?block=blk-…` はデスクトップと同一に解決し、該当ブロックへスクロールする。

### 4.5 サイドパネル = 閲覧専用ボトムシート

- `SidePanel`(340/320px 右パネル)は非描画。ヘッダ右端のパネルボタン(タブアイコン)に代えて、本文右下の FAB(44×44px、border-radius 999px、bg `var(--pr-a)`、影 plans/08 の `shadow-pop`)をタップすると**ボトムシート**(height 60dvh、上角 radius 16px、上端にグラバー 36×4px、backdrop 同 §4.3、下スワイプ/backdrop タップで閉じる)が開く(決定)。
- シート内タブは **チャット履歴 / 注釈 / 情報 の 3 タブのみ**(決定)。メモ・図表・リソースタブは M1 モバイルでは提供しない。各タブ本体はデスクトップのタブコンポーネント(`ChatTab` / `AnnotationsTab` / `InfoTab`)を `readOnly` prop 付きで再利用し、`readOnly=true` のとき入力欄・作成/編集/削除ボタンを非描画にする(決定。各タブ所有ファイル 1a/1b/1c に `readOnly` prop を追加する)。
- コンポーネント: `apps/web/src/components/viewer/mobile/MobileBottomSheet.tsx`(新規。本書所有)。
- 読書位置自動保存・読書時間計測(viewer-shell §8)はモバイルでも同一に動作させる(決定)。

## 5. ライブラリ / ダッシュボードのモバイル縮退

### 5.1 ライブラリ(4a カードビュー)

- `LibraryShell`(`components/shell/LibraryShell.tsx`)のサイドバー(`SidebarNav`)は非描画とし、トップバーにハンバーガー(≡)を置いて §4.3 と同型の左ドロワーで表示する(決定。ナビゲーションは閲覧に伴う遷移のため許可)。
- カードグリッドは `grid-template-columns: 1fr` の **1 カラム**(決定)。カード幅 100%(−左右 padding 16px)。
- ビュー切替(カード⇄テーブル)・一括選択・新規保存フィルタ等の操作系は非描画。クイックフィルタ 5 種・並び替えドロップダウン・検索は閲覧系として提供する(決定)。
- テーブルビュー(1e)の URL で開かれた場合もカードビューを描画する(決定)。

### 5.2 ダッシュボード(1d)

- 多カラム構成を `flex-direction: column` の**縦積み 1 カラム**にする(決定)。セクション順はデスクトップの DOM 順のまま。カード類は幅 100%。操作系ボタン(あれば)は §1.2 の規則で非描画。

### 5.3 ステータスピルのドロップダウン(カード上)

- **モバイルではライブラリカード上の `StatusPill` を `interactive=true` にする**(決定)。デスクトップの 4a §「カード上では変更不可」の決定はデスクトップ限定とし、モバイルではビューアを開かずにステータス変更を完了できることを優先する(docs/00 受け入れ基準「モバイルで閲覧とステータス変更ができ」への対応)。挙動は plans/08 §5.2 の interactive 仕様+viewer-shell §4.2-4 と同一(6 値 Popover、楽観更新、`done` は 1g モーダル経由)。
- カードクリック(ビューアへ遷移)との衝突は、ピル上のタップで `stopPropagation` して回避する(決定)。

## 6. 実装方式(決定)

1. **同一コンポーネント+CSS**: モバイル専用ページ・専用ルートは作らない。既存コンポーネントに `@media (max-width: 767px)` のスタイルを追加する。
2. **メディアクエリ**を用いる(コンテナクエリは用いない。決定。理由: 縮退の判定対象が常にビューポートであり、JS 側 `useMediaQuery` と判定源を一致させるため)。Tailwind では `max-md:`(`--breakpoint-md: 768px` 前提)を使用する。
3. **操作系の条件レンダリング**: §1.2 の非表示は `useMediaQuery` の結果(`isMobile`)による JS 条件レンダリングで行う(CSS 非表示は不可。決定)。SSR ではデスクトップとして描画し、ハイドレーション後に `isMobile` を反映する(初回フレームのちらつきは許容。決定)。
4. 新規ファイル: `hooks/useMediaQuery.ts` / `components/viewer/toc/TocDrawer.tsx` / `components/viewer/mobile/MobileBottomSheet.tsx`。既存タブ 3 種への `readOnly` prop 追加は各所有ファイル(1a/1b/1c)側の変更として扱う。

## 7. 受け入れ基準(AC-00-04・PW-22 対応)

E2E は viewport **390×844**(PW-22。plans/12)で実行する。

- [ ] 390×844 で `/papers/[itemId]`・`/library`・`/dashboard` に**横スクロールが発生しない**(`document.documentElement.scrollWidth <= 390`)
- [ ] ビューアが訳文モード固定で開き、段落タップで対訳ポップが開閉する
- [ ] 目次ドロワー・ボトムシート(チャット履歴/注釈/情報の閲覧)が開閉できる
- [ ] ビューアヘッダとライブラリカードのステータスピルから 6 値(planned / up_next / reading / done / reread / on_hold)を変更でき、`done` は読了フローモーダル(1g)を経由する
- [ ] **取り込み操作が要求されない**(取り込み導線・空状態の取り込み CTA が一切表示されない)
- [ ] 注釈作成・チャット送信・語彙追加・記事再生成・一括操作の UI が DOM に存在しない
- [ ] 768px 以上ではデスクトップレイアウト(`min-width:1200px` 水平スクロール許容)に戻る

以上。
