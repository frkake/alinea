# 訳読 / YAKUDOKU — 論文読解ワークベンチ

> **要旨**: 英語論文を日本語で深く読むための研究者向けツール。対訳を軸に、翻訳・注釈・読解チャット・語彙学習・外部リソース参照までを1つのワークベンチにまとめる。取り込みは Chrome / Edge 拡張機能に一本化し、ビューアは「訳文 / 対訳 / 原文 / PDF / 記事」の5表示モードを持つ。本 README はビジョンと機能の概観であり、詳細仕様は [docs/](docs/README.md) を参照。

## 1. ビジョン

論文を読む時間の大半は「英文を解釈する負荷」と「読んだ内容を思い出す負荷」に費やされる。訳読は、原文に忠実な日本語訳と根拠付きの AI 対話を1つの画面に置くことで、読解そのものに集中できる環境をつくる。

判断基準となるプロダクト原則(詳細: [docs/00-product.md](docs/00-product.md)):

| 原則 | 内容 |
|---|---|
| P1 忠実性 | 翻訳もチャット回答も原文を正とし、根拠(セクション/式/図のアンカーチップ)を必ず示す。AI 生成物には「AI生成」「論文外の知識」「推測」ラベルを付け、本文由来と区別する |
| P2 1アクション・段階的提示 | 取り込みは URL 1つ。追加直後にカードが現れ、読める部分から段階的に翻訳が進む(書誌→アブスト訳→本文) |
| P3 黙って壊れない | 翻訳失敗ブロックは原文+理由を表示。位置を失った注釈は「未配置」として残す。勝手に消さない・勝手に状態を変えない |
| P4 位置アンカーと検索 | 読んだ痕跡(ハイライト・メモ・チャット)はすべて論文上の位置に紐づき、あとから横断検索でヒット源明示つきで見つけられる |
| P5 ロックインしない | メモ/注釈/チャットは Markdown、書誌は BibTeX/CSV、全量 JSON でいつでも持ち出せる |
| P6 読書の邪魔をしない・管理を強要しない | AI 機能は呼んだときだけ現れる。進捗は読む行動から育つ。ステータス自動変更はせず提案のみ。ゲーミフィケーションはしない。統計は控えめ |
| P7 著作権を設計で守る | 全文翻訳は取り込んだ本人の閲覧に限る。共有は「書誌+要約+許可したメモ」まで。図表転載はライセンスが許す場合のみ+クレジット自動付記 |

## 2. 目指すもの

- 英語論文を、図表・数式・引用構造の文脈を失わずに日本語で読めるようにする
- 読みながら AI と対話し、疑問点をその場で深掘りできるようにする(回答には必ず原文の根拠を付ける)
- 論文中の気になる単語・コロケーション・イディオムを文脈ごと保存し、英語学習として復習できるようにする
- 読んだ論文をブログ風の「記事」として読み返し、記憶に定着させる
- 進捗・優先度・締切を「読む行動」から自然に育て、管理作業を強要しない
- 輪読会・研究室単位で、コレクションの共有リンクにより読書リストを共有できるようにする
- メモ・注釈・書誌をいつでも標準形式で持ち出せるようにする(ロックインしない)

## 3. 主な機能

確定デザインの画面インベントリに対応する。画面 ID((1a)(4d) など)は各 docs から参照される。

### 3.1 論文ビューア — 5つの表示モード

ユーザーが最も長くいる画面。左=目次 / 中=本文 / 右=サイドパネルの3カラム構成。

| モード | 概要 | 画面 |
|---|---|---|
| 訳文 | 1カラムの日本語訳。段落ホバーの「対」ボタンやキー `t` で対訳ポップ(原文を確認、「訳がおかしい?」→再翻訳導線) | (1b) |
| 対訳 | 段落単位の左右2カラム+「段落対応⇄」 | (1a) |
| 原文 | 構造化された英語原文 | — |
| PDF | 原紙面。構造化ビューと page+bbox で相互リンク(「同期: p.5 ≒ §2.2」) | (2a) |
| 記事 | 論文をブログ風に再構成して読み返す表示モード(§3.8) | (1h) |

- サイドパネルは排他6タブ: **チャット / メモ / 注釈 / 図表 / リソース / 情報**
- 注釈は4色(重要/疑問/アイデア/用語)+コメント。位置を失った注釈は「未配置」として残す
- 選択メニュー: 4色ドット+「コメント / ✦AIに質問 / 語彙に追加 / コピー」
- 図表参照クリックでその場ポップオーバー(図+原文/訳キャプション+「図の位置へ移動→」「✦この図を説明」)(1c)
- 数式ブロックに「✦この式を説明」「LaTeXをコピー」
- 前回位置の保存と「続きから」再開バナー、ダークモード対応(1c)
- 品質レベルは **A / B の2段階**をバッジで常時明示: A=LaTeX から完全構造化 / B=PDF 由来。決定: 旧 A〜D の4段階は廃止し C/D は B に統合(理由: ユーザーに意味のある差は「完全構造化か否か」の1点であり、確定デザインも2値バッジのみを持つ。OCR は将来検討)

詳細: [docs/04-viewer.md](docs/04-viewer.md)

### 3.2 読解チャット

- 論文ごとの専用チャット(サイドパネル)。スレッド切替可(1a)
- セクション・段落・図表・数式を選択して質問。回答には ¶(段落)粒度までの根拠チップを付け、クリックで原文位置へ
- 「AI生成」バッジと「論文外の知識」ボックスで本文由来と区別。免責文を固定表示
- 定型チップ5種: 「3行要約 / 初心者向け解説 / 貢献と限界 / 実験設定の整理 / 実装の要点」
- 回答の「↑メモに保存」「再生成」「コピー」

詳細: [docs/05-chat.md](docs/05-chat.md)

### 3.3 日本語訳の生成

- 論文全体の日本語訳。図・表・数式・引用・脚注・参考文献への参照(プレースホルダ)を維持
- 段階的提示: 書誌→アブスト訳→本文の順に、読める部分から先に届く。目次に節ごとの翻訳進捗を表示
- 未翻訳の付録は「開くと翻訳します(オンデマンド)」
- 翻訳スタイル既定(自然訳/直訳)、専門用語の一貫した訳語管理、段落単位の再翻訳
- テキスト生成 LLM は **OpenAI GPT / Anthropic Claude / Google Gemini / DeepSeek を用途別に切替可能**(運営既定+ユーザー設定+BYOK)

詳細: [docs/03-translation.md](docs/03-translation.md)

### 3.4 ライブラリ・ダッシュボード

- ステータス6値: 読む予定 / すぐ読む / 読んでいる / 読んだ / あとで再読 / 保留
- ライブラリはカード(4a)/テーブル(1e)の2ビュー。クイックフィルタ(すべて/未読/途中/読了/要再確認)、属性フィルタ(ステータス/タグ/コレクション/品質/年)、「この条件を保存」による保存フィルタ、複数選択の一括操作(ステータス変更/タグ追加/コレクションへ)
- 優先度(高/中/低)・締切・読了日・読書時間・理解度の記録。ステータスは自動変更せず提案のみ(既定)
- ダッシュボード(1d): 続きを読む / すぐ読むキュー(ドラッグ順序、積みすぎ警告)/ 締切 / 最近追加(パイプライン進捗+✦3行要約+タグ提案)/ 控えめな統計
- 読了フロー(1g): 理解度1〜5・重要度(低/中/高)・ひとことメモ・「✦要約をメモに保存」「記事モードで読み返す→」。すべてスキップ可

詳細: [docs/06-library.md](docs/06-library.md)

### 3.5 コレクションと共有

- 順序付きコレクション(4b): 締切+残日数、読了進捗、担当者割当(輪読会の担当表示)、行ごとの進捗/理解度
- 共有ページ(4c): `yakudoku.app/c/{token}` の閲覧専用リンク。アカウント不要・noindex。書誌+✦要約+許可したメモのみを公開し、「共有ページにメモを含める」トグルで制御
- 決定: v1 の共有経路はコレクション共有ページのみ。記事の公開・限定公開・コメントは v2 検討(理由: 確定デザインに記事公開 UI が存在せず、記事は個人の読み返しビューとして設計されている)

詳細: [docs/06-library.md](docs/06-library.md)

### 3.6 語彙帳 — 論文文脈から育てる英語学習

訳語統一のための用語集とは別物の、第一級の英語学習機能(4d)。

- 単語 / コロケーション / イディオム の3分類。ビューアの選択メニュー「語彙に追加」で文脈センテンスごと保存
- 語義・語源・解釈のしかた・覚えるコツを保存時に文脈から AI 自動生成(編集可)
- 忘却曲線ベースの SRS 復習: 復習期チップ、次の復習表示、自己評価「まだあやしい」/「✓覚えた」
- 「原文で見る→」で保存元の文脈センテンスへジャンプ

詳細: [docs/11-vocabulary.md](docs/11-vocabulary.md)

### 3.7 リソース — 論文外の学習材料の添付

論文に GitHub実装 / YouTube動画 / 発表スライド(PDF) / 解説記事 をひも付ける(5a、サイドパネル「リソース」タブ)。

- URL 貼り付けで種類を自動判定し、タイトル・サムネイルを自動取得
- arXiv 由来の論文は公式実装を自動検出し提案カードで提示(「公式実装」バッジ)
- 種類別メタ(言語・スター・更新 / 再生時間 / 枚数 / 読了目安)+§参照チップを埋め込めるひとことメモ

詳細: [docs/12-resources.md](docs/12-resources.md)

### 3.8 概要図と記事モード

- **全体概要図**: 課題→提案→結果の3カードフロー図。構造化図データから SVG を決定的にレンダリングし、版管理(「AI生成 · 版 2」)・指示付き書き直し・SVG ダウンロードに対応
- **詳細解説図・挿絵**: 記事に添える追加2枚程度のラスター図は画像生成 API(GPT / Gemini / Grok から切替可)で生成。概要図のラスター生成モードも設定で選択可能。決定: 全体概要図は SVG を正とする(理由: 確定デザインが版管理と SVG ダウンロードを持ち、テキスト崩れのない決定的な出力が必要なため。ラスター生成は画像内テキストが破綻しうるので補助用途に限定)
- **記事モード**(1h): 訳文・メモ・チャット履歴を素材に AI がブログ風に再構成する表示モード。独立エディタではない
  - ブロックホバーで「✦書き直し指示 / 再生成 / 根拠を表示」。レベル/テンプレート選択、ヘッダの「✦指示つき再生成」
  - 根拠チップ(§/表)と原文引用ブロック、免責「元の論文とは別物です — 根拠チップから原文へ」
  - 図の転載時はライセンスバッジ+クレジット自動付記。出典ブロックは自動挿入・削除不可
  - 「議論したい点」を疑問ハイライトから ✦AI 構成

詳細: [docs/07-figures-and-articles.md](docs/07-figures-and-articles.md)

### 3.9 通知

ヘッダの通知アイコン(グリフ「◷」)+通知ポップオーバー(4a)。3種: 翻訳完了(「読み始める→」)/ ステータス変更提案(「変更する」「そのまま」)/ 締切リマインド。「すべて既読にする」。詳細: [docs/06-library.md](docs/06-library.md)

### 3.10 横断検索

ライブラリヘッダの検索ドロップダウン(1e)+専用の横断検索結果画面(4e)。ヒット源をバッジで明示(本文/訳文/メモ/チャット/記事)し、日英クロス検索(日本語→訳文、英語→原文)に対応。論文単位でグループ化し、源別に該当位置へ遷移。詳細: [docs/06-library.md](docs/06-library.md)

### 3.11 設定

8カテゴリ(4f): アカウント / 表示 / 翻訳 / 読書の計測と提案 / チャット / 通知 / エクスポート / ブラウザ拡張。

- 翻訳: スタイル既定、付録・表セルの自動翻訳制御、長大論文のセクション選択提案
- 計測と提案: 読書時間計測 ON/OFF、ステータス自動遷移=自動適用/提案する(既定)/提案しない
- エクスポート: 論文単位 Markdown(Obsidian 互換)/ BibTeX・CSV / JSON 一括
- LLM プロバイダ・モデル選択と BYOK(API キー)は「翻訳」「チャット」「アカウント」カテゴリ内に配置

### 3.12 Chrome / Edge 拡張機能 — 唯一の取り込み経路

取り込みは拡張機能(3a)に一本化する。アプリ内の「+追加」ボタン・取り込みダイアログは持たない。決定: 経路を1つに絞る(理由: 確定デザインの主要な設計判断。「ツールバー→保存の2クリック」で P2(1アクション)を満たし、アプリ側 UI を読むことに専念させる)。

- 保存前: 書誌プレビュー+ステータス3択(読む予定/すぐ読む/読んでいる)+タグ提案+コレクション+ひとことメモ、Enter で保存。LaTeX ソース有無を判定し「品質レベルA見込み」を表示
- 保存直後: パイプライン進捗(書誌✓→構造化✓→翻訳中%)
- 既にライブラリ: 進捗・前回位置・「続きから開く」
- 一般 PDF: 「このタブのPDFを送信」(private、書誌は推定)
- URL のみ送信し解析はサーバー側。Manifest V3、Chrome / Edge 対応

詳細: [docs/08-extension.md](docs/08-extension.md)

## 4. 対応入力

すべて拡張機能経由で取り込む。詳細: [docs/02-ingest.md](docs/02-ingest.md)

| 入力 | 経路 | 品質レベル |
|---|---|---|
| arXiv(最優先) | 論文ページ/PDF ページから拡張で URL 送信。HTML・LaTeX ソースを活用し、PDF だけに依存しない構造化 | **A**(LaTeX から完全構造化。数式・相互参照・図表・脚注を保持) |
| 一般の論文 PDF | 開いているタブから拡張で直接送信(private、書誌は推定) | **B**(PDF 由来) |

- メタデータ(DOI・タイトル・著者・年・カテゴリ)取得、重複検知と既存ライブラリへの統合、代表図のサムネイル自動選択
- OCR が必要な PDF は将来検討。arXiv 以外の HTML 論文サイト(OpenReview、ACL Anthology 等)への対応も将来検討

## 5. スタイル

落ち着いた紙のようなデザイン。色味少なめ。確定デザインのトークンが正([docs/09-nonfunctional.md](docs/09-nonfunctional.md))。

- **配色**: キャンバス `#E8E6DF` / 面 `#FBFAF7`・`#F4F3EF` / カード `#FFFFFF`。本文 `#1E2227`〜`#24272B`、補助 `#5B6067`・`#777B81`
- **アクセント4色切替**: スレートブルー `#3E5C76`(既定)/ 緑 `#4A6B57` / 紫 `#6E5A7E` / テラコッタ `#7A5C48`。淡容器 `--pr-as`(10%)・枠 `--pr-am`(32%)を導出
- **ダークモード**: 基調 `#181B20` 系。アクセントは明度を上げた対応色(例: `#3E5C76→#8FAECB`)に自動マップ
- **注釈4色**: 重要=アンバー `#C49432` / 疑問=ブルー `#5884AA` / アイデア=グリーン `#659471` / 用語=グレー `#82827E`(彩度低め)
- **書体4系統**: UI・見出し=IBM Plex Sans JP / 訳文本文=Noto Serif JP(明朝。設定でゴシックに切替可)/ 原文=Source Serif 4 / コード・ID=IBM Plex Mono
- **基準ビューポート**: デスクトップ 1440×900。モバイルは v1 では閲覧・ステータス変更のみ

## 6. 対象ユーザー

- 研究者(英語論文を日常的に読む。輪読会・研究室での共有を含む)
- 論文の実装者(手法の理解と公式実装・解説リソースの参照)
- 論文を英語学習の素材にしたい読者(語彙帳による復習)
