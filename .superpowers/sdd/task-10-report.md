**SHA-256 manifest approach:** `apps/api/src/alinea_api/static/katex/manifest.json` — generated by Python `hashlib.sha256` over each file, keyed by relative path.

**Font inlining strategy:**
- `.woff` and `.ttf` references stripped from CSS (`_FONT_OTHER_RE` regex removes fallback src entries)
- `.woff2` references replaced with `data:font/woff2;base64,...` inline data URIs
- Result: zero relative font path references in output HTML

**Note on `http://` in KaTeX JS:** The minified JS contains W3C namespace URI string literals (`http://www.w3.org/2000/svg`, `http://www.w3.org/1998/Math/MathML`). These are JS string constants, not network requests. Tests check `src=`/`href=` HTML attributes and CDN domain strings instead of raw `http://`.
